Years ago, as ChatGPT gained popularity, communities explored the idea of using AI as an [[using_ai_to_handle_customer_queries | FAQ personal assistant]] to automatically generate answers to common questions in Slack channels <a class="yt-timestamp" data-t="00:00:27">[00:00:27]</a>. This RAG (Retrieval Augmented Generation) implementation is now considered one of the most classic AI use cases <a class="yt-timestamp" data-t="00:00:41">[00:00:41]</a>. Even today, despite being a classic application, chat platform assistance remains one of the most crucial AI use cases for any business <a class="yt-timestamp" data-t="00:00:53">[00:00:53]</a>.

## Capabilities of AI in Chat Platforms

AI assistants, particularly in platforms like Slack, can offer a wide range of [[capabilities_of_ai_in_slack_chat_platforms | capabilities]] to enhance productivity:

*   **RAG Functionality**
    *   The ability to search a knowledge base (e.g., Confluence, Google Drive, Notion) to answer user questions with relevant information <a class="yt-timestamp" data-t="00:01:29">[00:01:29]</a> <a class="yt-timestamp" data-t="00:05:08">[00:05:08]</a>. This allows the AI to provide precise answers based on specific documents or data sources <a class="yt-timestamp" data-t="00:08:24">[00:08:24]</a>.
*   **Web Search**
    *   AI assistants can search the internet to find information not contained within their internal knowledge base <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a> <a class="yt-timestamp" data-t="00:06:08">[00:06:08]</a>. This is useful for retrieving up-to-date data like pricing or general knowledge <a class="yt-timestamp" data-t="00:09:08">[00:09:08]</a>.
*   **Custom Tools and Functions**
    *   AI assistants can invoke custom tools <a class="yt-timestamp" data-t="00:01:32">[00:01:32]</a>.
    *   They can fetch information from a URL, effectively performing web scraping <a class="yt-timestamp" data-t="00:06:12">[00:06:12]</a>.
    *   Custom functions can be added using the OpenAI specification for function calling, allowing the assistant to perform almost any desired action <a class="yt-timestamp" data-t="00:06:21">[00:06:21]</a>.
*   **Conversation Summarization**
    *   The AI can summarize conversations in specific channels <a class="yt-timestamp" data-t="00:01:34">[00:01:34]</a> <a class="yt-timestamp" data-t="00:06:16">[00:06:16]</a>. This is a highly important use case, as it saves users from reading through many messages and threads after being away <a class="yt-timestamp" data-t="00:10:52">[00:10:52]</a>.
*   **Other Actions**
    *   Image interpretation and generation <a class="yt-timestamp" data-t="00:05:54">[00:05:54]</a>.
    *   Providing the current date, which is important for time-sensitive questions, as LLMs often don't inherently know the current date <a class="yt-timestamp" data-t="00:05:59">[00:05:59]</a>.

## Integration and Customization

Platforms like Runar make it incredibly easy and affordable to create AI assistants for various chat platforms, including Slack, Discord, or Teams <a class="yt-timestamp" data-t="00:01:11">[00:01:11]</a>.

Key aspects of [[integrations_and_collaboration_in_ai_platforms | integration]] and customization include:

*   **Flexible Deployment**: Assistants can be invited to only specific channels in a workspace, limiting the exposure of chats and sensitive information <a class="yt-timestamp" data-t="00:01:39">[00:01:39]</a>.
*   **Chat Memory**: Each thread in a chat platform functions as a separate chat memory, similar to individual conversations in ChatGPT <a class="yt-timestamp" data-t="00:08:44">[00:08:44]</a>.
*   **Trigger Options**: Users can configure how the bot is triggered, such as by mentioning its name (e.g., "@run bear") or via specific keywords or emojis, ensuring it doesn't respond to every message <a class="yt-timestamp" data-t="00:07:33">[00:07:33]</a>.
*   **Model Selection**: Users can select from various models, including GPT <a class="yt-timestamp" data-t="00:05:12">[00:05:12]</a>.
*   **Instruction Customization**: Users can provide specific instructions to the AI agent, dictating its behavior and response formatting <a class="yt-timestamp" data-t="00:05:55">[00:05:55]</a>.
*   **Connecting External AI**: Beyond using base models, some platforms allow connecting existing AIs, including custom AI agents developed with tools like LangChain and LangServe, or custom AI developed with Python SDKs <a class="yt-timestamp" data-t="00:04:08">[00:04:08]</a>.
