## [[ai_for_lead_generation | Leveraging AI for Lead Generation]] and Nurturing

In digital marketing, the key objectives for agencies are reaching as many leads as possible, as fast as possible, and with high personalization <a class="yt-timestamp" data-t="00:03:23">[00:03:23]</a>.

### The Role of [[understanding_ai_agents | AI Agents]]
[[understanding_ai_agents | AI agents]] are particularly well-suited for personalized lead nurturing at scale <a class="yt-timestamp" data-t="00:03:45">[00:03:45]</a>. They can manage numerous conversations with leads who have provided their information, with the goal of guiding them towards actions like purchasing a product, scheduling an appointment, or booking an event <a class="yt-timestamp" data-t="00:03:49">[00:03:49]</a>. This need for lead nurturing is universal across diverse businesses, from solar panel companies to chiropractic offices <a class="yt-timestamp" data-t="00:04:02">[00:04:02]</a>.

## [[building_ai_agents_for_crm_platforms | Building an AI Lead Nurturing Agent for GoHighLevel]]

A specific AI agent for lead nurturing within GoHighLevel has been developed, leveraging LangChain, LangGraph, and Pinecone for Retrieval Augmented Generation (RAG) <a class="yt-timestamp" data-t="00:04:27">[00:04:27]</a>.

### Integration via Webhooks
The AI agent integrates with GoHighLevel via webhooks. By setting up a webhook URL in the GoHighLevel app, the agent can receive requests whenever inbound or outbound SMS messages occur <a class="yt-timestamp" data-t="00:06:21">[00:06:21]</a>. This allows the AI agent to handle conversations as a lead nurturing bot <a class="yt-timestamp" data-t="00:06:53">[00:06:53]</a>.

### Agent Configuration and Customization
The platform allows agencies to connect different [[building_ai_agents | AI agents]] to their various clients (sub-accounts within GoHighLevel) <a class="yt-timestamp" data-t="00:08:46">[00:08:46]</a>. This enables highly dynamic and customized lead nurturing:
*   **Per-Client Agents:** Agencies can create distinct agents for different client types (e.g., a solar panel company vs. a chiropractic office) <a class="yt-timestamp" data-t="00:09:02">[00:09:02]</a>.
*   **Custom Prompts and Models:** Each agent can have unique prompts, different models, and temperature settings <a class="yt-timestamp" data-t="00:09:07">[00:09:07]</a>.
*   **Lead-Specific Agents:** Automation workflows in GoHighLevel can apply specific tags to leads, determining which AI agent should interact with them <a class="yt-timestamp" data-t="00:09:58">[00:09:58]</a>. This means different ads leading to different nurturing strategies <a class="yt-timestamp" data-t="00:10:13">[00:10:13]</a>.

### Core Tools and Actions
The AI agent uses a LangGraph executable with various tools to interact with GoHighLevel and manage conversations <a class="yt-timestamp" data-t="00:10:49">[00:10:49]</a>. These tools include:
*   Adding tags to leads <a class="yt-timestamp" data-t="00:11:09">[00:11:09]</a>
*   Removing tags <a class="yt-timestamp" data-t="00:11:14">[00:11:14]</a>
*   Invoking generic webhooks for custom events <a class="yt-timestamp" data-t="00:11:16">[00:11:16]</a>
*   Fetching calendar availability <a class="yt-timestamp" data-t="00:11:27">[00:11:27]</a>
*   Canceling appointments <a class="yt-timestamp" data-t="00:11:37">[00:11:37]</a>
*   Booking appointments <a class="yt-timestamp" data-t="00:11:39">[00:11:39]</a>

Custom actions can be defined using natural language in the front end, instructing the Large Language Model (LLM) when to use specific tools <a class="yt-timestamp" data-t="00:11:57">[00:11:57]</a>. This allows for highly customized conversation flows based on unique agency needs <a class="yt-timestamp" data-t="00:12:22">[00:12:22]</a>.

### Dynamic Prompt Construction
When a text message arrives, the system fetches the relevant agent based on the contact's location and tags <a class="yt-timestamp" data-t="00:09:20">[00:09:20]</a>. A dynamic prompt is then constructed for the LangGraph executable, incorporating various elements for effective conversation <a class="yt-timestamp" data-t="00:12:40">[00:12:40]</a>:
*   **FAQs:** Retrieved from a vector database (Pinecone) using RAG to answer common questions <a class="yt-timestamp" data-t="00:12:57">[00:12:57]</a>.
*   **Extra Context:** Information about the business location (e.g., address, phone number) <a class="yt-timestamp" data-t="00:13:11">[00:13:11]</a>.
*   **Defined Actions:** The specific tools mentioned above <a class="yt-timestamp" data-t="00:13:18">[00:13:18]</a>.
*   **Date Information:** Helps prevent the LLM from hallucinating dates, crucial for brick-and-mortar businesses with scheduled visits <a class="yt-timestamp" data-t="00:13:23">[00:13:23]</a>.
*   **Time Zone:** Ensures accurate scheduling <a class="yt-timestamp" data-t="00:13:32">[00:13:32]</a>.
*   **Base Prompt:** Configured in the front end, defining the agent's goals and constraints <a class="yt-timestamp" data-t="00:13:37">[00:13:37]</a>.

This robust prompt enables the AI agent to generate appropriate text message responses and call necessary tools, such as [[using_ai_to_handle_customer_queries | booking appointments]] <a class="yt-timestamp" data-t="00:13:53">[00:13:53]</a>. Additional tools can also be added, such as requesting Google reviews after an appointment <a class="yt-timestamp" data-t="00:14:39">[00:14:39]</a>.
