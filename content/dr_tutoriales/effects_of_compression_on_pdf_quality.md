PDF compression aims to reduce the file's weight and size, making it easier to send via platforms like WhatsApp or email, and to save space on a computer <a class="yt-timestamp" data-t="00:00:04">[00:00:04]</a>.

## How Compression Impacts Quality

When a PDF file is compressed, a trade-off often occurs between file size and quality <a class="yt-timestamp" data-t="00:01:56">[00:02:00]</a>. The more a file is compressed, the lower its quality will be <a class="yt-timestamp" data-t="00:01:56">[00:02:00]</a>. This reduction in quality can affect various elements within the PDF, including images, videos, and text <a class="yt-timestamp" data-t="00:02:06">[00:02:10]</a>. Essentially, compression involves sacrificing some quality to achieve a smaller file size <a class="yt-timestamp" data-t="00:02:10">[00:02:13]</a>.

## Compression Options and Their Outcomes

Online tools like [[Using I Love PDF for file compression | I Love PDF]] offer different compression levels, allowing users to choose the desired balance between file size and quality <a class="yt-timestamp" data-t="00:01:52">[00:01:56]</a>:

*   **Extreme Compression** <a class="yt-timestamp" data-t="00:01:53">[00:01:53]</a>: This option achieves the smallest file size but results in the most significant loss of quality <a class="yt-timestamp" data-t="00:03:31">[00:03:33]</a>.
*   **Recommended Compression** <a class="yt-timestamp" data-t="00:01:54">[00:01:54]</a>: This level offers a good balance, significantly reducing file size while attempting to maintain acceptable quality <a class="yt-timestamp" data-t="00:02:32">[00:02:34]</a>.
*   **Low Compression** <a class="yt-timestamp" data-t="00:01:55">[00:01:55]</a>: This option results in less file size reduction but preserves more of the original quality.

### Case Study: PDF Compression Example

In a demonstration, an original PDF file, "Fati Case Stadis," weighed approximately 7.82 megabytes <a class="yt-timestamp" data-t="00:01:15">[00:01:18]</a>. Using the "Recommended Compression" option on [[Using I Love PDF for file compression | I Love PDF]], the file size was reduced to 1.76 megabytes <a class="yt-timestamp" data-t="00:02:57">[00:03:00]</a>, representing a 78% reduction <a class="yt-timestamp" data-t="00:02:55">[00:02:57]</a>. In this specific case, where the content was primarily text with some images, minimal quality loss was observed <a class="yt-timestamp" data-t="00:03:05">[00:03:12]</a>. The final size was 1.75 megabytes <a class="yt-timestamp" data-t="00:03:22">[00:03:23]</a>, demonstrating a successful compression of up to 75% depending on the quality <a class="yt-timestamp" data-t="00:03:25">[00:03:28]</a>. Had "Extreme compression" been chosen, the file size could have been reduced even further <a class="yt-timestamp" data-t="00:03:31">[00:03:33]</a>.
