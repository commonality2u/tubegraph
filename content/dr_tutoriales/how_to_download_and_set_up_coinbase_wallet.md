From: [[dr_tutoriales]] <br/> 

[[coinbase_wallet_overview_and_features | Coinbase Wallet]] is described as a secure application for storing cryptocurrencies and tokens <a class="yt-timestamp" data-t="00:01:03">[00:01:03]</a>. This tutorial focuses on its use on mobile phones <a class="yt-timestamp" data-t="00:01:19">[00:01:19]</a>.

## Availability and Download <a class="yt-timestamp" data-t="00:01:11">[00:01:11]</a>

[[coinbase_wallet_overview_and_features | Coinbase Wallet]] is available on:
*   Google Play Store (for Android devices) <a class="yt-timestamp" data-t="00:01:11">[00:01:11]</a>, <a class="yt-timestamp" data-t="00:02:23">[00:02:23]</a>
*   Apple App Store (for iPhone devices) <a class="yt-timestamp" data-t="00:01:12">[00:01:12]</a>, <a class="yt-timestamp" data-t="00:02:27">[00:02:27]</a>
*   As an extension for Google Chrome <a class="yt-timestamp" data-t="00:01:13">[00:01:13]</a>

To download on a mobile device, users should go to their respective app store (Google Play or App Store), search for "Coinbase Wallet," and click "download" <a class="yt-timestamp" data-t="00:02:20">[00:02:20]</a>.

## Initial Setup <a class="yt-timestamp" data-t="00:02:44">[00:02:44]</a>

Upon opening the downloaded application, users have the option to:
1.  **Enter with an existing wallet** <a class="yt-timestamp" data-t="00:02:44">[00:02:44]</a>
2.  **Create a new wallet** <a class="yt-timestamp" data-t="00:02:45">[00:02:45]</a>

### Terms of Service and Privacy Policy <a class="yt-timestamp" data-t="00:02:48">[00:02:48]</a>
Users must confirm and accept the terms of service and privacy policy to proceed <a class="yt-timestamp" data-t="00:02:48">[00:02:48]</a>.

### Set a Username <a class="yt-timestamp" data-t="00:02:53">[00:02:53]</a>
A username must be chosen, which will be visible to other [[coinbase_wallet_overview_and_features | Coinbase Wallet]] users if the profile is public <a class="yt-timestamp" data-t="00:02:53">[00:02:53]</a>.

### Choose Privacy Settings <a class="yt-timestamp" data-t="00:03:07">[00:03:07]</a>
Users can select their desired privacy type:
*   **Public**: Allows other [[coinbase_wallet_overview_and_features | Coinbase Wallet]] users to search for their username and send cryptocurrencies <a class="yt-timestamp" data-t="00:03:10">[00:03:10]</a>. This is generally recommended <a class="yt-timestamp" data-t="00:03:22">[00:03:22]</a>.
*   **Private**: Other wallets cannot find the user's profile <a class="yt-timestamp" data-t="00:03:17">[00:03:17]</a>.

### Protect Your Wallet <a class="yt-timestamp" data-t="00:03:27">[00:03:27]</a>
To [[how_to_secure_and_back_up_your_coinbase_wallet | protect]] the wallet, users can enable security features:
*   **Face ID/Touch ID**: For iPhones, this allows authentication using biometric data <a class="yt-timestamp" data-t="00:03:30">[00:03:30]</a>.
*   **Password/PIN**: For other mobile devices, a password or PIN can be set <a class="yt-timestamp" data-t="00:03:33">[00:03:33]</a>.

## Backup and Recovery <a class="yt-timestamp" data-t="00:03:43">[00:03:43]</a>

Users are prompted to [[how_to_secure_and_back_up_your_coinbase_wallet | back up]] their [[coinbase_wallet_overview_and_features | Coinbase Wallet]] <a class="yt-timestamp" data-t="00:03:43">[00:03:43]</a>. It is crucial to understand that losing the recovery phrase means losing access to the account <a class="yt-timestamp" data-t="00:03:48">[00:03:48]</a>.

### Recovery Phrase <a class="yt-timestamp" data-t="00:04:09">[00:04:09]</a>
A secret recovery key, typically 12 words, is provided <a class="yt-timestamp" data-t="00:04:09">[00:04:09]</a>.

### Backup Options <a class="yt-timestamp" data-t="00:04:11">[00:04:11]</a>
Users can choose to [[how_to_secure_and_back_up_your_coinbase_wallet | back up]] via:
*   **iCloud backup** <a class="yt-timestamp" data-t="00:04:12">[00:04:12]</a>
*   **Manual backup**: This involves taking a screenshot of the words or copying them to a secure location like a notepad <a class="yt-timestamp" data-t="00:04:14">[00:04:14]</a>.

### Verifying the Backup <a class="yt-timestamp" data-t="00:04:36">[00:04:36]</a>
After backing up, the user must re-enter the recovery words in the correct order to confirm the backup <a class="yt-timestamp" data-t="00:04:36">[00:04:36]</a>.

## Wallet Interface Overview <a class="yt-timestamp" data-t="00:04:49">[00:04:49]</a>

Once set up, the main wallet screen displays:
*   Current balance (initially zero for a new wallet) <a class="yt-timestamp" data-t="00:04:51">[00:04:51]</a>
*   The chosen username <a class="yt-timestamp" data-t="00:04:56">[00:04:56]</a>
*   Sections for [[managing_cryptocurrencies_and_nfts_in_coinbase_wallet | cryptocurrencies and NFTs]] associated with the wallet <a class="yt-timestamp" data-t="00:05:00">[00:05:00]</a>

A menu at the bottom of the screen typically includes:
*   **Wallet**: The main section for [[managing_cryptocurrencies_and_nfts_in_coinbase_wallet | managing cryptocurrencies]] <a class="yt-timestamp" data-t="00:05:07">[00:05:07]</a>.
*   **Internet/Browser**: To access online platforms and decentralized applications (DApps) <a class="yt-timestamp" data-t="00:10:07">[00:10:07]</a>.
*   **Settings**: For configuring account details, security, and other preferences <a class="yt-timestamp" data-t="00:05:09">[00:05:09]</a>.

## Connecting with Coinbase Account <a class="yt-timestamp" data-t="00:07:06">[00:07:06]</a>

Users can connect their [[coinbase_wallet_overview_and_features | Coinbase Wallet]] to their main Coinbase exchange account <a class="yt-timestamp" data-t="00:07:06">[00:07:06]</a>. This allows for easier [[sending_and_receiving_cryptocurrencies_with_coinbase_wallet | transfers]] of funds between the two platforms <a class="yt-timestamp" data-t="00:07:14">[00:07:14]</a>, <a class="yt-timestamp" data-t="00:08:26">[00:08:26]</a>. Once connected, all funds, including cryptocurrencies and cash, are linked between the [[coinbase_wallet_overview_and_features | Coinbase Wallet]] and the Coinbase platform <a class="yt-timestamp" data-t="00:08:30">[00:08:30]</a>.
