The [[payments_tracker | customer payments tracker]] is a simple template designed to help users keep track of all their customer payments in one centralized location <a class="yt-timestamp" data-t="00:00:04">[00:00:04]</a>.

## Template Sections

The tracker is divided into distinct sections:

*   **Summary Section** (Left Side) <a class="yt-timestamp" data-t="00:00:09">[00:00:09]</a>
*   **Customer Section** (Right Side) <a class="yt-timestamp" data-t="00:00:20">[00:00:20]</a>
*   **Overview Section** (Right Side) <a class="yt-timestamp" data-t="00:00:21">[00:00:21]</a>
*   **Transaction Section** (Right Side) <a class="yt-timestamp" data-t="00:00:21">[00:00:21]</a>

### Summary Section

This section provides a quick overview of payment statistics:
*   Total amount of payment due <a class="yt-timestamp" data-t="00:00:11">[00:00:11]</a>
*   Payment amount received <a class="yt-timestamp" data-t="00:00:12">[00:00:12]</a>
*   Difference between due and received amounts <a class="yt-timestamp" data-t="00:00:14">[00:00:14]</a>
*   Percentage of total payments received <a class="yt-timestamp" data-t="00:00:16">[00:00:16]</a>

### Customer Section

The customer section allows for setting up an entire customer database <a class="yt-timestamp" data-t="00:00:24">[00:00:24]</a>.
*   **Subsections**: It is divided into 'Regular', 'Seasonal', and 'Occasional' subsections <a class="yt-timestamp" data-t="00:00:28">[00:00:28]</a>.
    *   Users can edit existing subsections <a class="yt-timestamp" data-t="00:00:33">[00:00:33]</a>.
    *   Users can add new subsections <a class="yt-timestamp" data-t="00:00:37">[00:00:37]</a>.
*   **Customer Management**:
    *   Each subsection lists individual customers <a class="yt-timestamp" data-t="00:00:42">[00:00:42]</a>.
    *   The total payment amount due for each customer is displayed <a class="yt-timestamp" data-t="00:00:48">[00:00:48]</a>.
    *   The due amount can be edited <a class="yt-timestamp" data-t="00:00:50">[00:00:50]</a>.
    *   Customer names can be edited <a class="yt-timestamp" data-t="00:00:56">[00:00:56]</a>.
    *   New customers can be added <a class="yt-timestamp" data-t="00:01:00">[00:01:00]</a>.
    *   Customers can be deleted <a class="yt-timestamp" data-t="00:01:05">[00:01:05]</a>.

### Overview Section

The overview section displays aggregated payment information once the customer database and transactions are entered <a class="yt-timestamp" data-t="00:01:10">[00:01:10]</a>:
*   Total payment received <a class="yt-timestamp" data-t="00:01:15">[00:01:15]</a>
*   Payment amount due <a class="yt-timestamp" data-t="00:01:16">[00:01:16]</a>
*   Percentage of payment received <a class="yt-timestamp" data-t="00:01:19">[00:01:19]</a>

### Transaction Section

This section is where individual payment transactions are recorded and updated <a class="yt-timestamp" data-t="00:01:22">[00:01:22]</a>.
*   **Data Fields**:
    *   Date of transaction <a class="yt-timestamp" data-t="00:01:25">[00:01:25]</a>
    *   Description of the transaction <a class="yt-timestamp" data-t="00:01:25">[00:01:25]</a>
    *   Customer name (selectable from the database) <a class="yt-timestamp" data-t="00:01:27">[00:01:27]</a>
    *   Amount of payment <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a>
*   **Document Attachment**: A document section is available to attach specific invoices related to the customer transactions <a class="yt-timestamp" data-t="00:01:35">[00:01:35]</a>.
*   **Automatic Updates**: Once transaction details are filled, the received payment amounts in the overview and summary sections automatically update <a class="yt-timestamp" data-t="00:01:41">[00:01:41]</a>.

## Customization
The entire template is customizable to user requirements <a class="yt-timestamp" data-t="00:01:54">[00:01:54]</a>.
