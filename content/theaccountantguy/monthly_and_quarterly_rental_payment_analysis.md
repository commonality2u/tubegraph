The House Rent Payments Tracker is a template designed to help users keep track of their house rental payments <a class="yt-timestamp" data-t="00:00:06">[00:00:06]</a>. It provides various views and sections to manage tenant information, property details, and financial summaries related to rent <a class="yt-timestamp" data-t="00:00:10">[00:00:10]</a>.

## Overview of the House Rent Payments Tracker

The tracker's summary section offers four different views <a class="yt-timestamp" data-t="00:00:10">[00:00:10]</a>. Users can add basic information such as tenants, who can be categorized under different property classifications like "apartment" and "duplex" <a class="yt-timestamp" data-t="00:00:13">[00:00:13]</a>. New tenants can be added using an "ADD tenant" button and then placed under the respective property category <a class="yt-timestamp" data-t="00:00:27">[00:00:27]</a>.

## Key Features for Analysis

The template includes several sections crucial for detailed analysis of rental payments:

### Property-Level Financials
The "property section" displays different property classifications <a class="yt-timestamp" data-t="00:00:46">[00:00:46]</a>. Under each property, users can view the tenant's name and key financial information, including:
*   Total rent amount <a class="yt-timestamp" data-t="00:00:54">[00:00:54]</a>
*   Rent amount that has been paid <a class="yt-timestamp" data-t="00:00:55">[00:00:55]</a>
*   Balance rent amount that is due <a class="yt-timestamp" data-t="00:00:57">[00:00:57]</a>
*   Advanced rent that has been paid so far <a class="yt-timestamp" data-t="00:00:58">[00:00:58]</a>

### [[tracking_rental_history_and_transaction_summaries | Rental History]] Tracking
The "rental history" view provides a record of all tenants who have paid rent, based on the transactions booked in the system <a class="yt-timestamp" data-t="00:01:04">[00:01:04]</a>.

### Overall Financial Summary
A dedicated summary section shows:
*   Total amount due <a class="yt-timestamp" data-t="00:01:14">[00:01:14]</a>
*   Amount that has been paid <a class="yt-timestamp" data-t="00:01:15">[00:01:15]</a>
*   Outstanding rent amount <a class="yt-timestamp" data-t="00:01:16">[00:01:16]</a>
*   Percentage of rent collected <a class="yt-timestamp" data-t="00:01:17">[00:01:17]</a>

### Detailed Transaction Views
The transaction section offers three distinct views, in addition to a view of all transactions <a class="yt-timestamp" data-t="00:01:20">[00:01:20]</a>:
*   **Rent Paid**: Displays all transactions for which rent has been paid, along with respective tenants and amounts <a class="yt-timestamp" data-t="00:01:24">[00:01:24]</a>.
*   **Rent Due**: Shows information for rent that is due and has not yet been paid <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a>. Clicking a checkbox indicates payment, moving the transaction to the "Rent Paid" category <a class="yt-timestamp" data-t="00:01:36">[00:01:36]</a>.

### [[quarterly_financial_summaries | Quarterly Summaries]]
The "quarterly summary" view allows users to see rental payments and collections, as well as the balance due for every month within a specific quarter <a class="yt-timestamp" data-t="00:01:41">[00:01:41]</a>. This feature aids in [[monthly_view_and_performance_tracking | monthly]] and quarterly performance tracking of rental income.

## Adding and Managing Payments
Users can quickly add new rental payments by clicking a designated button within the template <a class="yt-timestamp" data-t="00:01:50">[00:01:50]</a>.
