## How Layer 2s Scale Ethereum

[[the_evolution_and_future_of_ethereum_layer_2_solutions | Layer 2 solutions]] primarily work by executing transactions off-chain and then batching them up at the end of a block to publish back onto Layer 1 [[ethereums_role_and_infrastructure_in_financial_markets | Ethereum]] <a class="yt-timestamp" data-t="03:48:00">[03:48:00]</a>. This method significantly increases throughput while maintaining [[ethereums_role_and_infrastructure_in_financial_markets | Ethereum's]] security <a class="yt-timestamp" data-t="03:06:00">[03:06:00]</a>.

### Modular Architecture
Mantle employs a modular architecture, which contrasts with monolithic chains like [[ethereums_role_and_infrastructure_in_financial_markets | Ethereum]] (especially before upgrades like Dencun) and [[comparison_of_ethereum_with_other_blockchain_platforms | Solana]] <a class="yt-timestamp" data-t="03:55:00">[03:55:00]</a>. In a monolithic chain, execution, data availability, consensus, and finality all occur on the same layer using the same block. While this works with low traffic, surging activity leads to high gas fees <a class="yt-timestamp" data-t="03:55:00">[03:55:00]</a>.

Mantle, however, builds in a "Lego-like" modular way:
*   **Execution**: Handled on Mantle <a class="yt-timestamp" data-t="03:26:00">[03:26:00]</a>.
*   **Consensus/Finality**: Handled on [[ethereums_role_and_infrastructure_in_financial_markets | Ethereum]] <a class="yt-timestamp" data-t="03:31:00">[03:31:00]</a>.
*   **Data Availability (DA)**: Handled on Mantle DA, powered by EigenLayer's technology <a class="yt-timestamp" data-t="03:32:00">[03:32:00]</a>.

This modularity allows for greater agility and adaptability to the latest technological advancements <a class="yt-timestamp" data-t="03:42:00">[03:42:00]</a>. For example, Mantle V2 "Teutonic" upgrade only required changing specific parts of the code, unlike the lengthy and complex upgrades of monolithic chains <a class="yt-timestamp" data-t="03:42:00">[03:42:00]</a>.

### Leveraging EigenLayer for Security
Mantle utilizes EigenLayer, a premium restaking layer, to bootstrap security and liquidity <a class="yt-timestamp" data-t="03:32:00">[03:32:00]</a>. EigenLayer allows protocols to "rent" security from [[ethereums_role_and_infrastructure_in_financial_markets | Ethereum]] stakers themselves, enabling new projects to start with significant security from day one without needing to build their own market cap <a class="yt-timestamp" data-t="03:37:00">[03:37:00]</a>. This approach abstracts away the challenge of bootstrapping security and liquidity, enabling developers to focus on solving core problems <a class="yt-timestamp" data-t="03:51:00">[03:51:00]</a>.

## User Experience and Adoption Challenges

Despite the technical advancements, [[the_evolution_and_future_of_ethereum_layer_2_solutions | Layer 2s]] face user experience [[challenges_and_strategies_for_scaling_cryptocurrency_protocols | challenges and strategies for scaling cryptocurrency protocols]] such as liquidity fragmentation and the complexity of bridging assets <a class="yt-timestamp" data-t="04:40:00">[04:40:00]</a>. Users often prefer to pay higher fees on the mainnet to avoid these complexities <a class="yt-timestamp" data-t="04:40:00">[04:40:00]</a>.

Solutions being developed include:
*   **Account Abstraction**: Smart wallets that hide the underlying technical complexities from users, allowing them to interact with [[ethereums_role_and_infrastructure_in_financial_markets | Ethereum]] via [[the_evolution_and_future_of_ethereum_layer_2_solutions | Layer 2s]] without realizing it <a class="yt-timestamp" data-t="04:56:00">[04:56:00]</a>.
*   **Intent-based Protocols**: These protocols allow users to specify their desired end result, and the system automatically finds the optimal routing across various blockchains, abstracting away the need for users to choose networks or bridge assets manually <a class="yt-timestamp" data-t="04:51:00">[04:51:00]</a>. Mantle's Eco Fund actively invests in such projects <a class="yt-timestamp" data-t="04:51:00">[04:51:00]</a>.
