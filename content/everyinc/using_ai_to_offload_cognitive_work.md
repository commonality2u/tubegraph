## Modes of AI Assistance

AI can function in several modes to offload cognitive tasks:

### Co-pilot Mode
This mode involves a human making a conscious decision in real-time to interact with AI for assistance, then proceeding with their work <a class="yt-timestamp" data-t="02:01:01">[02:01:01]</a>. It helps individuals be more effective, representing a classic form of symbiosis or intelligence augmentation <a class="yt-timestamp" data-t="03:03:03">[03:03:03]</a>. ChatGPT is a primary example of this mode <a class="yt-timestamp" data-t="02:01:01">[02:01:01]</a>.

### Delegation Mode
In this mode, a task is truly offloaded to AI, aiming for output consistency that eliminates the need for constant human review <a class="yt-timestamp" data-t="02:36:00">[02:36:00]</a>. This allows for significant time and cost savings on burdensome tasks or the scaling of currently unscalable processes <a class="yt-timestamp" data-t="03:10:03">[03:10:03]</a>.

### Ad Hoc Delegation (Future Development)
Currently, there's a gap between real-time co-pilot assistance and full delegation <a class="yt-timestamp" data-t="03:53:00">[03:53:00]</a>. The ability to delegate larger, more significant subtasks to AI on the fly is not yet fully realized <a class="yt-timestamp" data-t="04:03:00">[04:03:00]</a>. However, it is anticipated that this gap will close in the near future as AI agents become more capable <a class="yt-timestamp" data-t="04:28:00">[04:28:00]</a>.

## Practical [[applications_of_ai_in_automating_tasks | Applications of AI in Automating Tasks]]

### Coding Assistance
An example of [[using_ai_tools_for_creative_projects | using AI tools for creative projects]] and offloading cognitive work is using ChatGPT to build a React app <a class="yt-timestamp" data-t="00:03:00">[00:03:00]</a>. For someone unfamiliar with React, ChatGPT can:
*   Explain the project structure and various frameworks (Redux, Saga JS) <a class="yt-timestamp" data-t="0:29:13">[0:29:13]</a>.
*   Provide commands to print out file structures, overcoming common installation or syntax hurdles <a class="yt-timestamp" data-t="0:32:16">[0:32:16]</a>.
*   Debug code by analyzing what was attempted and explaining why it's not working <a class="yt-timestamp" data-t="0:35:20">[0:35:20]</a>.
*   Generate new code and instructions for modifications, even assisting with styling (CSS) <a class="yt-timestamp" data-t="0:35:32">[0:35:32]</a>.

This process, which would typically take days in a pre-ChatGPT era, can be reduced to a few hours, largely due to the AI writing most of the code and guiding the user through unfamiliar syntax and best practices <a class="yt-timestamp" data-t="0:37:20">[0:37:20]</a>. This saves 80-90% of the time and eliminates much of the frustration associated with getting stuck on common, undifferentiated problems <a class="yt-timestamp" data-t="0:38:23">[0:38:23]</a>.

### Writing Assistance
[[role_of_ai_in_creative_work | Role of AI in creative work]] is evident in writing tasks. AI excels at:
*   **Summarization**: Much of creative work, like programming, writing, and decision-making, involves summarizing <a class="yt-timestamp" data-t="10:49:00">[10:49:00]</a>. AI can quickly summarize complex ideas or lengthy texts, providing a foundation for original work <a class="yt-timestamp" data-t="13:59:00">[13:59:00]</a>.
*   **Outlining**: AI can take messy collections of notes, quotes, and ideas and organize them into coherent outlines, often pointing out obvious structures that a human might overlook due to being too close to the problem <a class="yt-timestamp" data-t="15:20:00">[15:20:00]</a>.
*   **Expression and Metaphor**: AI can help articulate thoughts, offering various ways to express an idea or generate metaphors until the exact desired phrasing is found <a class="yt-timestamp" data-t="16:16:00">[16:16:16]</a>.
*   **Critique and Feedback**: AI can critique drafts of emails, introductions, or other written content, providing valuable feedback for refinement <a class="yt-timestamp" data-t="19:15:00">[19:15:00]</a>.

While AI assists across the writing process, it is often used for "microtasks," with the final published content remaining a unique human creation <a class="yt-timestamp" data-t="16:50:00">[16:50:00]</a>.

### Information Gathering and Decision Making
For factual queries and decision support, tools like Perplexity provide significant assistance:
*   **Concrete Question Answering**: Perplexity is designed for accurate answers to specific questions, acting as a more direct rival to traditional search engines <a class="yt-timestamp" data-t="58:13:00">[58:13:00]</a>.
*   **Collating Complex Information**: It can rapidly collate and present detailed information from disparate sources, such as car trim levels and their features, which would be extremely time-consuming to research manually <a class="yt-timestamp" data-t="59:11:00">[59:11:00]</a>.
*   **"Good Enough" Accuracy**: While not always 100% accurate, Perplexity often provides information that is "good enough for practical purposes," enabling users to make informed decisions without exhaustive fact-checking <a class="yt-timestamp" data-t="01:01:34">[01:01:34]</a>.

This capability saves immense time and can act as a reliable alternative to editorial products like Wirecutter, by allowing users to ask any specific question and receive a relevant answer <a class="yt-timestamp" data-t="01:02:45">[01:02:45]</a>. This represents a new standard for immediately answering fact-based questions <a class="yt-timestamp" data-t="01:03:03">[01:03:03]</a>.

### Diagram and Patent Application Creation
AI can also assist with technical tasks like generating diagrams for patent applications. By describing an application's structure, AI can:
*   Suggest appropriate syntax (e.g., Mermaid, Graphviz) for diagrams <a class="yt-timestamp" data-t="50:25:00">[50:25:00]</a>.
*   Generate the diagram syntax, which can then be rendered in an external tool <a class="yt-timestamp" data-t="50:52:00">[50:52:00]</a>.
*   Iteratively refine the diagram based on feedback, even re-starting conversations if the AI gets confused by too many syntax changes <a class="yt-timestamp" data-t="51:25:00">[51:25:00]</a>.

This process provides a maintainable, structured language representation of complex systems that can be updated more easily than freehand drawings <a class="yt-timestamp" data-t="53:32:00">[53:32:00]</a>.
