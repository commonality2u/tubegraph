## AI as a Tool, Not an Agent

Language models are increasingly viewed not as entities with their own agency, but as sophisticated calculators <a class="yt-timestamp" data-t="00:04:00">[00:04:00]</a>. From a technical perspective, a language model is fundamentally a function that predicts a probability distribution <a class="yt-timestamp" data-t="04:59:00">[04:59:00]</a>. While AI tools can be given "anthropomorphic packaging" that makes them seem to have will or intent, they are ultimately statistical models <a class="yt-timestamp" data-t="05:04:00">[05:04:00]</a>.

Despite this, the design and "packaging" of AI tools can accidentally diminish user agency <a class="yt-timestamp" data-t="05:38:00">[05:38:00]</a>. For example, image generation tools like DALL-E find it difficult to produce "ugly" images because they are designed to produce aesthetically pleasing results, thereby constraining the user's creative search space <a class="yt-timestamp" data-t="06:37:00">[06:37:00]</a>, <a class="yt-timestamp" data-t="06:53:00">[06:53:00]</a>. This highlights that while the underlying AI technology is a mathematical object, its interface can either empower or disempower users <a class="yt-timestamp" data-t="07:11:00">[07:11:00]</a>, <a class="yt-timestamp" data-t="07:19:00">[07:19:00]</a>.

## AI Models and Their Implications

Modern AI models operate based on complex mathematical principles. One key concept is **embeddings**:
*   **Embeddings** are lists of numbers that quantitatively summarize the semantic content of text or images <a class="yt-timestamp" data-t="10:51:00">[10:51:00]</a>.
*   They represent data points as coordinates in a high-dimensional space, where closer points signify similar meanings <a class="yt-timestamp" data-t="11:41:00">[11:41:00]</a>.
*   For instance, an embedding model can recognize that "The Eiffel Tower is in Paris" and "The president of France is blah" are semantically closer than "Notion is a tool company" <a class="yt-timestamp" data-t="11:20:00">[11:20:00]</a>.
*   This numerical representation allows for mathematical manipulation, enabling creative transformations, such as adjusting the shade of a color or the "emotion" of a generated image <a class="yt-timestamp" data-t="13:28:00">[13:28:00]</a>, <a class="yt-timestamp" data-t="19:15:00">[19:15:00]</a>.

An example of [[using_ai_tools_for_creative_projects | using AI tools for creative projects]] and leveraging embeddings is a tool called "Varna" (a personal experiment) <a class="yt-timestamp" data-t="14:45:00">[14:45:00]</a>. Varna allows users to describe an image by adding and subtracting images and text <a class="yt-timestamp" data-t="15:20:00">[15:20:00]</a>. It uses a model like CLIP (Contrastive Language-Image Pre-training) which describes both text and image inputs within the same embedding space <a class="yt-timestamp" data-t="15:45:00">[15:45:00]</a>. This means a user can mix concepts, for example, combining a French flag image and a butterfly image to generate a butterfly with French flag colors <a class="yt-timestamp" data-t="17:02:00">[17:02:00]</a>. It's akin to having a "slider" for concepts or ideas in image generation, allowing a user to explore the continuous space between two inputs, such as transforming a headshot into a vectorized drawing or modulating an image's emotion from happy to angry <a class="yt-timestamp" data-t="18:44:00">[18:44:00]</a>, <a class="yt-timestamp" data-t="19:15:00">[19:15:00]</a>.

This approach reveals that models internally learn human-recognizable and semantically useful concepts <a class="yt-timestamp" data-t="20:17:00">[20:17:00]</a>. When viewed this way, with "dials and numbers" to manipulate, language models feel less like agents and more like controllable tools <a class="yt-timestamp" data-t="20:42:00">[20:42:00]</a>.

One of the most common applications of AI in creative work is programming help <a class="yt-timestamp" data-t="40:08:00">[40:08:00]</a>. While tools like Co-Pilot are excellent for micro-level code suggestions within an existing file, ChatGPT is more suitable for open-ended problems, such as learning new libraries or understanding how to approach complex data processing challenges <a class="yt-timestamp" data-t="40:29:00">[40:29:00]</a>. By providing detailed context about the problem, the user can receive better and more tailored suggestions from the AI <a class="yt-timestamp" data-t="44:09:00">[44:09:00]</a>.

## Prompting Strategies for Creativity

Two main approaches to prompting AI exist:
1.  **Prompt Engineering:** Used for robust, reliable applications (like those shipped to millions of users in products), requiring extensive testing, evaluation, and techniques like "few-shot prompting" and structured outputs <a class="yt-timestamp" data-t="35:50:00">[35:50:00]</a>, <a class="yt-timestamp" data-t="36:05:00">[36:05:00]</a>.
2.  **Scripting/Iterating:** More casual, iterative interaction, common when talking directly to tools like ChatGPT <a class="yt-timestamp" data-t="36:26:00">[36:26:00]</a>. Users can start with simple prompts and refine them through follow-up questions until the desired result is achieved <a class="yt-timestamp" data-t="37:06:00">[37:06:00]</a>. This approach encourages experimentation and learning the tool's capabilities through direct interaction <a class="yt-timestamp" data-t="37:50:00">[37:50:00]</a>.

Key prompting techniques include:
*   **Providing more context:** The more context an AI tool has about the user's goals, the better its suggestions will be <a class="yt-timestamp" data-t="00:08:00">[00:08:00]</a>, <a class="yt-timestamp" data-t="44:09:00">[44:09:00]</a>.
*   **Iterative refinement:** Instead of trying to perfect the first prompt, users can start broad and gradually narrow down the request by asking follow-up questions or re-generating responses <a class="yt-timestamp" data-t="43:42:00">[43:42:00]</a>, <a class="yt-timestamp" data-t="46:01:00">[46:01:00]</a>.
*   **"Chain of Density" / "Chain of Thought":** For higher quality or specific output properties (e.g., concision, detail), instruct the model to produce multiple drafts, critically review its own previous drafts, and then output a final, refined version <a class="yt-timestamp" data-t="01:06:36:00">[01:06:36:00]</a>. This is a variant of "Chain of Thought" prompting, where the AI is asked to write out its thinking steps <a class="yt-timestamp" data-t="01:08:28:00">[01:08:28:00]</a>.
*   **Manipulating conversational history:** In environments like the OpenAI Playground, users can insert fake chat history to "force the AI's hand" and make it more likely to perform certain actions or bypass typical guardrails <a class="yt-timestamp" data-t="01:12:58:00">[01:12:58:00]</a>.

The [[adaptive_use_of_ai_for_personal_and_creative_tasks | adaptive use of AI for personal and creative tasks]], coupled with strategic prompting, allows for greater precision and collaborative work with AI, making it a valuable asset in modern creative processes.
