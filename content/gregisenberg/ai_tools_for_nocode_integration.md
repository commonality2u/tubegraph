## Building Workflows with Gumloop

A Gumloop workflow is a series of steps where data is passed from one "node" to another, enabling the creation of powerful AI-powered workflows that connect directly with user data <a class="yt-timestamp" data-t="00:02:31">[00:02:31]</a>.

The platform is often compared to no-code automation tools like Zapier and Make due to its numerous integrations <a class="yt-timestamp" data-t="00:03:04">[00:03:04]</a>.

### Key Components:
*   **Nodes:** Users add nodes onto an empty canvas to build a workflow <a class="yt-timestamp" data-t="00:02:59">[00:02:59]</a>.
*   **Integrations:** Gumloop offers connections to various platforms, including Slack, Airtable, Outlook, Notion, Reddit, and Gmail <a class="yt-timestamp" data-t="00:03:09">[00:03:09]</a>. The true power emerges when this data is combined with AI steps <a class="yt-timestamp" data-t="00:03:20">[00:03:20]</a>.
*   **AI Steps:** These steps are powered by Large Language Models (LLMs) and include:
    *   **Ask AI:** Similar to asking ChatGPT a question, but allows plugging in any model, including those deployed on Azure <a class="yt-timestamp" data-t="00:03:45">[00:03:45]</a>.
    *   **Data Extraction:** Users can specify a schema (e.g., amount, date) for AI to extract structured data from content like receipts <a class="yt-timestamp" data-t="00:04:19">[00:04:19]</a>.
    *   **Categorization, Summarization, Scoring:** Other nuanced [[function_calling_and_tool_integration_in_ai_apps | AI tooling]] available <a class="yt-timestamp" data-t="00:04:43">[00:04:43]</a>.
    *   **Video and Image Analysis:** Advanced AI capabilities for multimedia content <a class="yt-timestamp" data-t="00:04:45">[00:04:45]</a>.

The combination of LLM steps with user data allows for the creation of fully AI-powered products that would typically require an engineer to build <a class="yt-timestamp" data-t="00:04:50">[00:04:50]</a>.

## Advanced Integration and Automation Features

Gumloop offers several features that distinguish it and enhance its integration capabilities:

### Subflows
Subflows allow an entire workflow to be used as a single node within another workflow <a class="yt-timestamp" data-t="00:07:08">[00:07:08]</a>. This provides the reusability and cleanliness of a function in programming, making workflows extensible and shareable among team members <a class="yt-timestamp" data-t="00:07:13">[00:07:13]</a>.

### API Triggering (Webhooks)
Any Gumloop workflow can be treated like an API, triggered via a webhook from an external product based on an event <a class="yt-timestamp" data-t="00:06:44">[00:06:44]</a>. This means users can [[integrating_ai_tools_in_building_saas_startups | build a SaaS product]] and have Gumloop perform the backend work, charging users for access to complex functionalities like PDF processors or contract analyzers <a class="yt-timestamp" data-t="00:24:43">[00:24:43]</a>.

### Loop Mode
Loop mode enables workflows to process thousands of events simultaneously by iterating the automation over a list of inputs <a class="yt-timestamp" data-t="00:13:14">[00:13:14]</a>. Gumloop manages the infrastructure, handling concurrent runs and rate limits <a class="yt-timestamp" data-t="00:15:26">[00:15:26]</a>.

### Custom Nodes
Users can build their own integrations by defining custom nodes. By providing API documentation to an AI-powered builder, Gumloop can generate the necessary code for a new integration, making it accessible to semi-technical users without needing an engineering degree <a class="yt-timestamp" data-t="00:31:08">[00:31:08]</a>. These custom nodes are then available to everyone in a shared workspace <a class="yt-timestamp" data-t="00:32:13">[00:32:13]</a>.

### Interfaces
Gumloop allows users to create simple, user-friendly interfaces on top of complex workflows <a class="yt-timestamp" data-t="00:21:24">[00:21:24]</a>. This hides the intricate flowchart from end-users, enabling co-workers to utilize powerful tools without understanding the underlying mechanics <a class="yt-timestamp" data-t="00:23:23">[00:23:23]</a>.

### Chrome Extension
A Chrome extension node allows workflows to start by scraping content from the user's screen or performing other actions like screenshots <a class="yt-timestamp" data-t="00:34:40">[00:34:40]</a>. This enables the creation of no-code Chrome extensions that leverage Gumloop's automation capabilities <a class="yt-timestamp" data-t="00:36:58">[00:36:58]</a>.
