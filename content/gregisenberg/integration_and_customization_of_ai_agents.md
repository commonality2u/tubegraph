## AI Agents vs. Workflows

While Lindy.AI's interface might resemble a workflow automation platform like Zapier, it operates as an AI agent under the hood <a class="yt-timestamp" data-t="00:03:30">[00:03:30]</a> <a class="yt-timestamp" data-t="00:06:50">[00:06:50]</a>. The key difference is that AI agents can "chat" and interpret natural language prompts within each step, rather than requiring separate AI steps for specific tasks <a class="yt-timestamp" data-t="00:03:42">[00:03:42]</a> <a class="yt-timestamp" data-t="00:04:04">[00:04:04]</a>. This allows for greater flexibility; for example, a user can simply instruct an agent to "summarize the meeting you just had in bullet points" without needing to pre-configure a specific AI summarization step <a class="yt-timestamp" data-t="00:03:59">[00:03:59]</a>. Agents are also designed to recover from mistakes <a class="yt-timestamp" data-t="00:04:29">[00:04:29]</a>.

## Customization and Control

[[personalization_and_customization_of_ai_designs | Customization]] is integral to Lindy's functionality:
*   **Prompt-based configuration:** Each field within an agent's step is essentially a prompt, allowing users to speak in natural language to define what the agent should do <a class="yt-timestamp" data-t="00:03:42">[00:03:42]</a>.
*   **Human-in-the-Loop:** Users can easily insert a human for confirmation at any step where an AI agent could potentially cause embarrassment. This acts as a training period, similar to onboarding an intern, with future features allowing the agent to learn from human feedback <a class="yt-timestamp" data-t="00:17:49">[00:17:49]</a> <a class="yt-timestamp" data-t="00:18:22">[00:18:22]</a>.
*   **LLM Selection:** Users can choose from various LLMs like Claude (3.5 as default), Gemini, and 01, depending on the agent's complexity and cost considerations <a class="yt-timestamp" data-t="00:21:43">[00:21:43]</a>.

## The Power of [[creating_ai_employees_and_agents | Agent Swarms]]

[[creating_ai_employees_and_agents | Agent swarms]] are a significant new development in AI agents <a class="yt-timestamp" data-t="00:10:20">[00:10:20]</a>. An agent swarm enables an AI agent to reliably, quickly, and in parallel perform a list of tasks that might otherwise cause long-term coherence issues or failures <a class="yt-timestamp" data-t="00:10:31">[00:10:31]</a>. For example, instead of one agent sending 500 personalized outreach emails slowly and potentially unreliably, an agent duplicates itself (like "Agent Smith" from *The Matrix*) to send one copy to each lead <a class="yt-timestamp" data-t="00:10:51">[00:10:51]</a> <a class="yt-timestamp" data-t="00:11:15">[00:11:15]</a>. A "swarm of swarms" can even be deployed, such as a daily meeting prep agent that deploys a swarm for each meeting, and within that, another swarm for each attendee to gather relevant information from Google Drive, emails, and LinkedIn <a class="yt-timestamp" data-t="00:11:25">[00:11:25]</a>.
