## The Vibe Coding Approach

Billy Howell, a self-taught developer who has successfully built and sold multiple Replit apps, explains his process for making a profit with [[vibe_coding_to_make_money_on_upwork | vibe coding]] <a class="yt-timestamp" data-t="01:03:00">[01:03:00]</a>, <a class="yt-timestamp" data-t="01:44:52">[01:44:52]</a>. The core secret is to find a buyer *before* building the app <a class="yt-timestamp" data-t="02:24:00">[02:24:00]</a>.

### Finding Opportunities on Upwork

Upwork is a prime starting point for finding clients who want to purchase custom applications <a class="yt-timestamp" data-t="02:39:00">[02:39:00]</a>. Instead of building something and then trying to find users, the strategy is to identify an existing problem or technology that an app can replace or improve <a class="yt-timestamp" data-t="02:49:00">[02:49:00]</a>.

*   **Search for existing technologies**: Look for jobs related to specific software like AirTable consulting <a class="yt-timestamp" data-t="02:59:00">[02:59:00]</a>. The goal is to identify situations where a custom app could be cheaper or better <a class="yt-timestamp" data-t="03:05:00">[03:05:00]</a>.
*   **Prioritize simplicity**: Focus on CRUD (Create, Read, Update, Delete) apps, which involve putting data into a database, pulling it, and visualizing it <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>. These are generally easier to build and deploy <a class="yt-timestamp" data-t="05:12:00">[05:12:00]</a>. Keywords like "automations" are also good to search for, as many can be turned into an app with a front end <a class="yt-timestamp" data-t="09:33:00">[09:33:00]</a>.
*   **Evaluate job postings**: Consider the complexity of integrations requested (e.g., Calendly, Zapier, AirTable, SMS functionalities) and the proposed [[pricing_strategies_for_ai_products | pricing]] <a class="yt-timestamp" data-t="04:27:00">[04:27:00]</a>, <a class="yt-timestamp" data-t="06:43:00">[06:43:00]</a>. A project priced at $125 is typically too low for a custom app <a class="yt-timestamp" data-t="06:52:00">[06:52:00]</a>, <a class="yt-timestamp" data-t="07:38:00">[07:38:00]</a>, while a $2,500 project for a custom workflow is more appealing <a class="yt-timestamp" data-t="10:05:00">[10:05:00]</a>, <a class="yt-timestamp" data-t="10:09:00">[10:09:00]</a>.
*   **Proposals**: Even if the listed price is low, it's possible to submit a proposal with a higher price <a class="yt-timestamp" data-t="07:03:00">[07:03:00]</a>. Upwork uses a "connects" system for applications, and proposals can be "boosted" to appear at the top <a class="yt-timestamp" data-t="07:09:00">[07:09:00]</a>.

### Building the Prototype with AI

Once a promising job is identified, the next step is to quickly build a prototype using AI tools.

*   **Product Requirements Document (PRD)**: Take the job description and feed it into ChatGPT, asking it to format the requirements as a PRD for an AI coding assistant, remaining agnostic to the tech stack but specifying UI libraries like ShadCN if desired <a class="yt-timestamp" data-t="11:14:00">[11:14:00]</a>, <a class="yt-timestamp" data-t="12:22:00">[12:22:00]</a>. A PRD tells an engineer all the necessary features and data types <a class="yt-timestamp" data-t="11:51:00">[11:51:00]</a>.
*   **Replit for rapid development**: Replit (specifically Replit V2) is recommended due to its ability to go from idea to MVP with minimal friction <a class="yt-timestamp" data-t="12:45:00">[12:45:00]</a>. It handles package management, installations, deployment, and testing <a class="yt-timestamp" data-t="12:52:00">[12:52:00]</a>. Replit's AI agent, powered by Claude 3.7, can scaffold out entire apps, building initial prototypes and even suggesting additional features like database integration <a class="yt-timestamp" data-t="17:04:00">[00:17:04]</a>, <a class="yt-timestamp" data-t="17:16:00">[00:17:16]</a>. It provides a running, functional starting point, overcoming "writer's block" for code <a class="yt-timestamp" data-t="18:16:00">[00:18:16]</a>.
*   **V0 for UI Enhancement**: For upgrading the UI, V0 can be used. Simply describe the desired page, and V0 can generate frontend code for a visually appealing mockup, which can then be pulled into Replit <a class="yt-timestamp" data-t="14:22:00">[14:22:00]</a>, <a class="yt-timestamp" data-t="14:55:00">[14:55:00]</a>. This is particularly useful if a client provides wireframes <a class="yt-timestamp" data-t="14:51:00">[14:51:00]</a>.

## Challenges and Considerations

*   **Complexity**: While AI tools are powerful, certain features can lead to complications. Payment processors, DocuSign integrations, and calendar/AI features (due to date formatting and time zones) tend to be more complex <a class="yt-timestamp" data-t="19:55:00">[19:55:00]</a>, <a class="yt-timestamp" data-t="20:17:00">[20:17:00]</a>. However, API integrations for basic data retrieval (GET calls) can be handled flawlessly <a class="yt-timestamp" data-t="20:55:00">[20:55:00]</a>.
*   **Quality Control**: While AI-generated apps are solid, they require "TLC" to ensure they are production-ready. A basic understanding of how code works is beneficial to anticipate how users might break the app <a class="yt-timestamp" data-t="19:20:00">[19:20:00]</a>.
*   **Hiring Help**: For the final 15% or for troubleshooting complex issues, Replit offers a "bounty" system where developers can be hired for an hourly rate or a fixed fee <a class="yt-timestamp" data-t="21:13:00">[21:13:00]</a>, <a class="yt-timestamp" data-t="22:09:00">[22:09:00]</a>. Paying $50 to troubleshoot can save hours and AI credits <a class="yt-timestamp" data-t="23:01:00">[23:01:00]</a>.

## Earnings and Business Models

*   **Upwork Earnings**: Top earners on Upwork can make over six figures annually <a class="yt-timestamp" data-t="23:47:00">[23:47:00]</a>. A solo developer can easily clear $5,000 to $10,000 per month by [[building_a_business_with_ai_tools | building custom apps]] or consulting on specific SaaS projects <a class="yt-timestamp" data-t="24:50:00">[24:50:00]</a>.
*   **Recurring Revenue**: Beyond the initial project fee, developers can charge for ongoing support, future feature requests, and hosting costs <a class="yt-timestamp" data-t="07:53:00">[07:53:00]</a>, <a class="yt-timestamp" data-t="38:40:00">[38:40:00]</a>. This positions the developer as a "dev agency" for their clients <a class="yt-timestamp" data-t="38:45:00">[38:45:00]</a>.
*   **Sales Beyond Upwork**: Referrals are a strong source of projects <a class="yt-timestamp" data-t="28:02:00">[28:02:00]</a>. Another strategy is to directly contact business owners, inquire about their most annoying tasks or expensive SaaS subscriptions, and then offer a custom app as a solution <a class="yt-timestamp" data-t="28:24:00">[28:24:00]</a>.
*   **Consulting vs. SaaS**: This approach is essentially [[human_versus_ai_services | consulting]], utilizing AI to build custom solutions <a class="yt-timestamp" data-t="28:50:00">[28:50:00]</a>. It's often contrasted with [[creating_ai_employees_for_startups | building a SaaS product]], which can be a "nightmare money pit" due to significant marketing costs and the need for strong distribution <a class="yt-timestamp" data-t="29:05:00">[29:05:00]</a>. While SaaS offers asymmetric bets, custom services de-risk projects by securing payment upfront <a class="yt-timestamp" data-t="34:57:00">[34:57:00]</a>.
    *   **Framework for Business Models**:
        1.  **Easy Mode**: Getting paid for a service <a class="yt-timestamp" data-t="34:09:00">[34:09:00]</a>.
        2.  **Medium Mode**: [[Using AI to build SaaS startups | Building B2B software]] <a class="yt-timestamp" data-t="34:16:00">[34:16:00]</a>.
