## Understanding AI Agents on Lindy.AI

Lindy.AI functions like a workflow automation platform, but under the hood, each process is an AI agent <a class="yt-timestamp" data-t="00:03:30">[00:03:30]</a>. This means users can interact with steps in plain English, and the AI maintains coherence across multiple steps <a class="yt-timestamp" data-t="00:03:42">[00:03:42]</a>. Agents are designed to recover from mistakes <a class="yt-timestamp" data-t="00:04:29">[00:04:29]</a>.

### Agent Swarms
An agent swarm involves sending a list of tasks to an AI agent, which then duplicates itself to perform these tasks reliably, quickly, and in parallel <a class="yt-timestamp" data-t="00:11:15">[00:11:15]</a>. This overcomes limitations of single AI agents, which can fail or become unreliable over long-term, repetitive tasks <a class="yt-timestamp" data-t="00:10:44">[00:10:44]</a>. For example, instead of one agent sending 500 personalized emails sequentially and potentially failing, an agent swarm duplicates the agent for each lead, ensuring more reliable and faster outreach <a class="yt-timestamp" data-t="00:10:51">[00:10:51]</a>.

## Getting Started with AI Agents

To effectively [[automating_workflows_using_ai|automate workflows using AI]], consider the following:

*   **Start Simple**: Begin with personal assistance use cases like meeting scheduling or recording <a class="yt-timestamp" data-t="00:15:07">[00:15:07]</a>. Lindy.AI provides hundreds of templates <a class="yt-timestamp" data-t="00:15:26">[00:15:26]</a>, including a "personal assistant starter pack" <a class="yt-timestamp" data-t="00:15:36">[00:15:36]</a>.
*   **Iterative Building**: Start small and gradually add more steps and complexity to the agents <a class="yt-timestamp" data-t="00:18:48">[00:18:48]</a>.
*   **Human in the Loop**: If an AI agent could cause embarrassment, it's advisable to include a human in the loop, especially in the initial stages <a class="yt-timestamp" data-t="00:18:02">[00:18:02]</a>. Think of it as training an intern; eventually, the agent can learn from feedback and correct itself <a class="yt-timestamp" data-t="00:18:18">[00:18:18]</a>.
*   **Choosing the Right LLM**: For most tasks, Claude 3.5 (default on Lindy.AI) or Gemini Flash are good starting points <a class="yt-timestamp" data-t="00:21:43">[00:21:43]</a>. For more complex or research-intensive tasks, more expensive models like O1 might be necessary <a class="yt-timestamp" data-t="00:21:58">[00:21:58]</a>.
