## Case Study: Guilty Chef (Recipe Directory)

Guilty Chef is a website that aggregates recipes from various restaurants worldwide <a class="yt-timestamp" data-t="00:06:02">[00:06:02]</a>. The founder, a UX/UI designer, created this as a side project to explore using AI for recipe [[automating_content_creation_with_ai | content creation]] <a class="yt-timestamp" data-t="00:04:50">[00:04:50]</a>.

### AI-Powered Content and Design
The website's content, including descriptions, prep times, cook times, ingredients, directions, and FAQs, is entirely [[automating_content_creation_with_ai | automatically generated]] using ChatGPT and Zapier <a class="yt-timestamp" data-t="00:06:47">[00:06:47]</a>. Even the images are [[creating_and_manipulating_aigenerated_art | AI-generated]] <a class="yt-timestamp" data-t="00:06:37">[00:06:37]</a>. The founder simply inputs a restaurant name and a menu item (e.g., "McDonald's Big Mac"), and an entire page is generated within a minute, fully designed and ready <a class="yt-timestamp" data-t="00:09:10">[00:09:10]</a>.

### SEO and Schema
A key aspect of Guilty Chef's success is the use of schema, which is structured data that Google uses to understand a website's content <a class="yt-timestamp" data-t="00:07:09">[00:07:09]</a>. For recipe websites, "recipe schema" defines properties like cook time, cooking method, nutritional value, and steps <a class="yt-timestamp" data-t="00:08:44">[00:08:44]</a>. AI can write an entire schema for a website, which can then be plugged into the site <a class="yt-timestamp" data-t="00:09:01">[00:09:01]</a>. Implementing schema correctly helps a site rank higher, even above major competitors like Trip Advisor, by presenting information to Google in the cleanest, easiest-to-understand format <a class="yt-timestamp" data-t="01:15:15">[01:15:15]</a>. Many large sites neglect proper schema implementation, providing a significant advantage for those who utilize it <a class="yt-timestamp" data-t="01:15:50">[01:15:50]</a>.

### Growth and Monetization
Guilty Chef saw traffic grow from zero in March to potentially exceeding 100,000 monthly visits by December, all with minimal time investment (about a week of full-time work initially, then a few hours a month) <a class="yt-timestamp" data-t="01:11:50">[01:11:50]</a>.

*   **Premium Products:** Charging for recipe packs (e.g., $2.99 for 10 recipes) <a class="yt-timestamp" data-t="01:11:23">[01:11:23]</a>.
*   **Sponsorships/Partnerships:** With restaurants or brands selling ingredients <a class="yt-timestamp" data-t="01:18:28">[01:18:28]</a>.
*   **Automated Affiliate Links:** A future opportunity could involve automatically linking ingredients to brands that would bid for placement <a class="yt-timestamp" data-t="01:19:08">[01:19:08]</a>.
*   **B2B Software:** Selling insights or software to restaurants based on consumer traffic data <a class="yt-timestamp" data-t="02:08:52">[02:08:52]</a>.
*   **Enhanced User Experience:** Continuously improving the user experience, such as customizing recipes for specific diets or family preferences, to become a go-to tool <a class="yt-timestamp" data-t="01:29:28">[01:29:28]</a>.

## Case Study: Best Dubai (Travel Directory)
Best Dubai is a travel directory created using the same AI-powered process as Guilty Chef, but focused on Dubai <a class="yt-timestamp" data-t="02:05:01">[02:05:01]</a>. The founder acquired the domain "bestdubai.com" for around $1,200, considering it a bargain due to Dubai's popularity as a tourist destination and high search volume for "best Dubai" keywords <a class="yt-timestamp" data-t="02:06:06">[02:06:06]</a>.

### AI-Enhanced Features
Similar to Guilty Chef, everything on Best Dubai is [[automating_content_creation_with_ai | AI-generated]] <a class="yt-timestamp" data-t="02:26:25">[02:26:25]</a>. This includes:
*   **Scores:** Claude AI helped write the algorithm to generate scores for places based on reviews and information <a class="yt-timestamp" data-t="02:46:26">[02:46:26]</a>.
*   **Best Times to Visit:** Information like peak busy periods is pulled directly from Google Maps data <a class="yt-timestamp" data-t="02:56:57">[02:56:57]</a>.
*   **Dynamic Menus:** A custom Chrome extension, created with Claude, scrapes up-to-date restaurant menu items and pricing from delivery services like Deliveroo and Talabat, addressing the common problem of outdated online menus <a class="yt-timestamp" data-t="03:07:07">[03:07:07]</a>.

### Rapid SEO Results
Within just 30 days of launch, Best Dubai began ranking on Google. For searches like "Parkers Dubai reviews," it appeared as the fourth organic link, competing with major platforms like Trip Advisor and Zomato, despite having no comparable marketing budget <a class="yt-timestamp" data-t="02:17:07">[02:17:07]</a>.

### Innovative Monetization and Branding
Beyond typical ad revenue, Best Dubai focuses on creating value:
*   **In-Content Advertising:** The goal is to develop a system where brands can bid for placement within the directory's content, such as ingredients in a recipe, without relying on intrusive Google Ads <a class="yt-timestamp" data-t="01:19:08">[01:19:08]</a>.
*   **Attribution & Backlinks:** Anytime a user visits a linked site from Best Dubai, the site automatically appends its own reference (e.g., `ref=gregsr`) to the URL <a class="yt-timestamp" data-t="01:49:24">[01:49:24]</a>. This allows businesses to see where their traffic is coming from, potentially leading to them proactively reaching out for partnerships or paid promotion <a class="yt-timestamp" data-t="01:50:13">[01:50:13]</a>.
*   **Custom Awards/Tags:** Creating proprietary categories like "Dubai J Best 100" and issuing physical awards to restaurants that achieve these distinctions helps build the brand and increase influence in the local market <a class="yt-timestamp" data-t="02:57:17">[02:57:17]</a>. This also encourages restaurants to embed the site's widget, generating valuable backlinks <a class="yt-timestamp" data-t="03:26:50">[03:26:50]</a>.

## How to Build an AI-Powered Directory

The process involves combining several tools to create a seamless, automated content generation and publishing pipeline.

### Essential Tools
*   **Webflow:** A visual web design and [[building_apps_with_ai_tools | development platform]] similar to Framer or Figma, but for building functional websites <a class="yt-timestamp" data-t="03:21:21">[03:21:21]</a>. It includes a Content Management System (CMS) to manage collections of listings (e.g., restaurants, malls, hotels) <a class="yt-timestamp" data-t="03:34:33">[03:34:33]</a>.
*   **Appify:** A platform offering various web scrapers for platforms like Instagram, TikTok, LinkedIn, and crucially, Google Maps <a class="yt-timestamp" data-t="03:04:49">[03:04:49]</a>. This allows for extracting rich data about places <a class="yt-timestamp" data-t="03:17:15">[03:17:15]</a>.
*   **Claude/ChatGPT:** AI language models used for [[automating_content_creation_with_ai | content generation]], data repurposing, and even algorithm development <a class="yt-timestamp" data-t="01:40:40">[01:40:40]</a>. Claude is favored for its more "human" output <a class="yt-timestamp" data-t="03:20:43">[03:20:43]</a>.
*   **Zapier:** An automation tool that connects different web applications, allowing for a workflow where data can be passed between Appify, Claude, and Webflow <a class="yt-timestamp" data-t="01:59:52">[01:59:52]</a>.

### Step-by-Step Process

#### 1. Data Acquisition with Appify
*   **Identify Niche:** Choose a niche for your directory (e.g., Montreal travel directory) <a class="yt-timestamp" data-t="03:11:10">[03:11:10]</a>.
*   **Source Data:** Instead of relying solely on AI, leverage "real-world knowledge" or existing curated content (e.g., popular TikToks from local bloggers) to find nuanced listings <a class="yt-timestamp" data-t="03:18:08">[03:18:08]</a>.
*   **Scrape with Appify:**
    1.  Use the Google Maps Business Scraper actor on Appify <a class="yt-timestamp" data-t="03:39:09">[03:39:09]</a>.
    2.  Find the Place ID for each location using Google's Place ID Finder <a class="yt-timestamp" data-t="03:49:15">[03:49:15]</a>.
    3.  Input the Place ID into Appify to scrape detailed information like address, city, images, and popular times (occupancy data) <a class="yt-timestamp" data-t="03:51:50">[03:51:50]</a>.
    4.  Extract the scraped data in JSON format <a class="yt-timestamp" data-t="04:02:40">[04:02:40]</a>.

#### 2. AI Content Generation with Claude
*   **Input Data to Claude:** Provide the JSON data from Appify to Claude <a class="yt-timestamp" data-t="04:19:19">[04:19:19]</a>. Claude can then interpret this data and generate insights or new content based on it <a class="yt-timestamp" data-t="04:27:07">[04:27:07]</a>.
*   **Craft Specific Prompts:**
    *   Define the desired output format for Claude using asterisks to delimit fields (e.g., `**Place Name**: Print the title of the restaurant`) <a class="yt-timestamp" data-t="04:57:51">[04:57:51]</a>.
    *   Be very specific with instructions within each mini-prompt, including desired language, style, or even data manipulation (e.g., changing image URL parameters for higher resolution) <a class="yt-timestamp" data-t="04:58:34">[04:58:34]</a>.
    *   Request specific metrics or content, like a scoring algorithm for food, service, and overall experience, or a summarized "best time for date night" <a class="yt-timestamp" data-t="05:12:08">[05:12:08]</a>.
*   **Refine Output:** Claude's output will be a single text block. Use a Python script within Zapier to parse this block and extract individual data points based on the asterisk delimiters <a class="yt-timestamp" data-t="05:54:19">[05:54:19]</a>.

#### 3. Automation with Zapier and Webflow
*   **Set up Webflow CMS:**
    1.  Create a "Collection" in Webflow's CMS (e.g., "Restaurants") <a class="yt-timestamp" data-t="03:40:08">[03:40:08]</a>.
    2.  Define fields within the collection (e.g., Name, Total Score, URL) <a class="yt-timestamp" data-t="03:49:55">[03:49:55]</a>.
    3.  Create a template page in Webflow for your listings (e.g., "Restaurant Template") <a class="yt-timestamp" data-t="01:10:00">[01:10:00]</a>.
    4.  Design the template, adding placeholders for the data you'll feed from the CMS <a class="yt-timestamp" data-t="01:10:10">[01:10:10]</a>.
    5.  Publish the Webflow site and collection for Zapier to recognize them <a class="yt-timestamp" data-t="01:10:47">[01:10:47]</a>.
*   **Connect with Zapier:**
    1.  **Trigger:** Set up a Google Sheet as the trigger. When a new row (containing a Place ID) is added, the Zap workflow begins <a class="yt-timestamp" data-t="01:45:01">[01:45:01]</a>.
    2.  **Appify Action:** Connect Appify to Zapier. The Place ID from the Google Sheet triggers the Google Maps Business Scraper on Appify <a class="yt-timestamp" data-t="01:46:16">[01:46:16]</a>.
    3.  **Code (Python) Action:** Use Zapier's "Code by Zapier" feature to run a Python script. This script fetches the raw JSON data URL from Appify's output and transforms it into a single JSON string, suitable for Claude <a class="yt-timestamp" data-t="01:52:18">[01:52:18]</a>.
    4.  **Claude/ChatGPT Action:** Send the processed JSON string to Claude with your detailed prompt, requesting specific data points in the desired format <a class="yt-timestamp" data-t="01:53:35">[01:53:35]</a>.
    5.  **Code (Python) Action (Parsing):** Another Python script in Zapier parses Claude's single text block output, extracting individual values based on the asterisk delimiters <a class="yt-timestamp" data-t="02:07:41">[02:07:41]</a>.
    6.  **Webflow Action:** Create a "Create Live Item" action for your Webflow CMS. Map the individual parsed data points from the Python script to the corresponding fields in your Webflow CMS collection <a class="yt-timestamp" data-t="02:08:49">[02:08:49]</a>.

This automated process allows for mass uploading of data, creating thousands of pages with minimal manual effort once set up <a class="yt-timestamp" data-t="01:45:31">[01:45:31]</a>.

> [!TIP] Troubleshooting AI Output
> If Claude's output includes unwanted text or formatting, refine your prompt. Use explicit instructions like "print only the score" or "no additional text" to ensure clean data for parsing <a class="yt-timestamp" data-t="01:16:37">[01:16:37]</a>. Special characters in the AI's output can also cause issues when trying to map data in Zapier or Webflow <a class="yt-timestamp" data-t="01:23:38">[01:23:38]</a>.
