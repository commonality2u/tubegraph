## Blake Anderson's Framework for Building Mainstream AI Products
Blake's framework for [[building_ai_apps_for_organic_growth | building AI apps for organic growth]] and making them mainstream involves a highly iterative process focusing on virality, smart distribution, and efficient development.

### 1. Ideation
The ideation process often starts by identifying what tools or features will perform well on social media <a class="yt-timestamp" data-t="14:20:00">[14:20:00]</a>. Blake focuses on [[ai_startup_incubation_and_ideas | AI startup incubation and ideas]] that redefine the value people get from a product, rather than just using TikTok algorithms for quick cash <a class="yt-timestamp" data-t="05:29:00">[05:29:00]</a>.

*   **Problem Identification:** Look for archaic or fragmented industries where AI can simplify processes (e.g., calorie tracking) <a class="yt-timestamp" data-t="05:05:00">[05:05:00]</a>.
*   **Inspiration:** Deliberate scrolling on social media and curating feeds around specific interests helps in understanding consumer content consumption <a class="yt-timestamp" data-t="36:06:00">[36:06:00]</a>.
*   **Market Analysis:** Using tools like Sensor Tower to assess the size of other players in the space <a class="yt-timestamp" data-t="35:44:00">[35:44:00]</a>.

### 2. Distribution
A key component is a highly iterative process of generating virality early on <a class="yt-timestamp" data-t="08:55:00">[08:55:00]</a>.
*   **Split Testing:** Constantly come up with new hooks and ways to position the product. Split test these across different videos and creators to assess views, engagement, and conversion rates <a class="yt-timestamp" data-t="09:00:00">[09:00:00]</a>.
*   **Influencer Marketing:** Instead of relying on marketplaces or agencies, Blake emphasizes developing personal relationships with influencers <a class="yt-timestamp" data-t="11:03:00">[11:03:00]</a>. This helps differentiate serious companies from the hundreds of daily inquiries influencers receive <a class="yt-timestamp" data-t="11:51:00">[11:51:00]</a>. Initially, one might need to pursue influencers aggressively to overcome the "cold start" problem, then treat them well to build lasting relationships <a class="yt-timestamp" data-t="12:41:00">[12:41:00]</a>.
*   **Targeting:** Focus on specific demographics and platforms (e.g., students on Instagram Reels, TikTok, Twitter, LinkedIn) <a class="yt-timestamp" data-t="06:59:00">[06:59:00]</a>.

### 3. Development
*   **Design First:** Start with wireframes, then hire professional designers (e.g., via Upwork) to convert them into polished UI <a class="yt-timestamp" data-t="14:29:00">[14:29:29]</a>.
*   **AI-Assisted Coding:** For those without capital or CTO connections, AI tools like ChatGPT can teach coding (e.g., Swift UI, React Native) and generate code snippets <a class="yt-timestamp" data-t="14:43:00">[14:43:00]</a>. Cursor with React Native is a current preference <a class="yt-timestamp" data-t="15:18:00">[15:18:00]</a>.
*   **Minimal Viable Product (MVP):** The goal is to get something functional out there quickly, focusing on the features most likely to go viral <a class="yt-timestamp" data-t="37:32:00">[37:32:00]</a>.

### 4. Monetization ([[monetization_strategies_for_mobile_apps | Monetization Strategies]])
Blake highly recommends Superwall for paywall management due to its split-testing capabilities and data provision <a class="yt-timestamp" data-t="24:16:00">[24:16:00]</a>.
*   **Pricing Strategy:** Blake deviates from the industry standard of optimizing for profit per user, instead focusing on maximizing the number of users (number of users raised to the N, where N > 1, multiplied by profit per user) <a class="yt-timestamp" data-t="24:45:00">[24:45:00]</a>. This strategy aims to dominate the market quickly and increase word-of-mouth, which is hard to measure but crucial <a class="yt-timestamp" data-t="25:38:00">[25:38:00]</a>.
*   **Paywall Placement:** The most standard and efficient practice is to take users through an onboarding flow (e.g., quiz, personal information collection) and then present the paywall at the end <a class="yt-timestamp" data-t="26:35:00">[26:35:00]</a>. While this can feel morally ambiguous as users have invested time, it is highly effective <a class="yt-timestamp" data-t="27:16:00">[27:16:00]</a>.

### 5. Naming Conventions
The decision to include "AI" in an app's name depends on the industry:
*   **Value-add:** Appending "AI" is valuable for innovating in relatively stagnant industries (e.g., Cal AI in calorie tracking), as it signifies sophistication <a class="yt-timestamp" data-t="20:27:00">[20:27:00]</a>.
*   **Caution:** Avoid "AI" in fields requiring 100% accuracy (e.g., medical documentation services), as it might cause skepticism about potential errors <a class="yt-timestamp" data-t="20:55:00">[20:55:00]</a>.
