## Case Studies

### Guilty Chef (Recipe Directory)
Guilty Chef was created as a side project to explore generating recipes using AI <a class="yt-timestamp" data-t="00:05:00">[00:05:00]</a>. The goal was to build a directory of recipes from various restaurants globally, as no comprehensive online resource existed <a class="yt-timestamp" data-t="00:06:07">[00:06:07]</a>.

Key aspects and results:
*   **Content Generation:** Thousands of recipe pages were automatically generated using AI, including descriptions, prep time, cook time, ingredients, directions, and FAQs <a class="yt-timestamp" data-t="00:06:33">[00:06:33]</a>. Even images are AI-generated <a class="yt-timestamp" data-t="00:06:37">[00:06:37]</a>.
*   **Schema Markup:** Critical for Google to understand website content, recipe schema was used to tell Google precisely what was on the page <a class="yt-timestamp" data-t="00:07:09">[00:07:09]</a>. AI can write the entire schema automatically <a class="yt-timestamp" data-t="00:09:01">[00:09:01]</a>. Implementing schema correctly allows for rich snippets in Google search results, providing direct answers to user queries <a class="yt-timestamp" data-t="00:15:06">[00:15:06]</a>.
*   **Ranking:** Achieved number one Google ranking for specific search terms like "Dum pan tiar" and "crispy chili mushrooms wager Mama" <a class="yt-timestamp" data-t="00:14:26">[00:14:26]</a>. Also ranked first for "Taco Supreme ingredients," appearing below Taco Bell but as the top rich snippet answer <a class="yt-timestamp" data-t="00:15:01">[00:15:01]</a>.
*   **Traffic & Monetization:** Started with zero traffic in March and is projected to exceed 100,000 monthly traffic by December, with almost no time spent on it after initial setup <a class="yt-timestamp" data-t="00:11:52">[00:11:52]</a>. Monetization includes a paid recipe generation feature <a class="yt-timestamp" data-t="00:11:23">[00:11:23]</a>. Future monetization could involve sponsorships with restaurants or ingredient brands, or a unique ad platform for ingredients <a class="yt-timestamp" data-t="00:18:28">[00:18:28]</a>. The long-term vision is for it to become a go-to recipe tool that learns family preferences <a class="yt-timestamp" data-t="00:20:00">[00:20:00]</a>.

## Core Workflow for Building AI-Powered Directories

The process involves leveraging [[Using AI tools for web development | several AI-powered tools]] to automate content creation and optimize for search engines. This can be replicated for any industry <a class="yt-timestamp" data-t="00:33:07">[00:33:07]</a>.

### 1. Webflow (Website Design & CMS)
*   **Purpose:** A visual development platform for building and designing websites <a class="yt-timestamp" data-t="00:33:21">[00:33:21]</a>. It functions as a content management system (CMS) <a class="yt-timestamp" data-t="00:34:36">[00:34:36]</a>.
*   **Features:** Create "collections" (e.g., restaurants, malls) to catalog listings <a class="yt-timestamp" data-t="00:34:56">[00:34:56]</a>. Design elements on a template page are dynamically connected to specific data fields in the CMS <a class="yt-timestamp" data-t="00:54:39">[00:54:39]</a>.
*   **Efficiency:** Utilizes components from sites like Flowbase or Flow Components, allowing users to copy and paste pre-designed sections directly into their Webflow project, saving design time <a class="yt-timestamp" data-t="00:34:40">[00:34:40]</a>.

### 2. Appify (Data Scraping)
*   **Purpose:** A platform offering various web scrapers for extracting data from sites like Instagram, TikTok, LinkedIn, and crucially, Google Maps <a class="yt-timestamp" data-t="00:37:09">[00:37:09]</a>.
*   **Google Maps Scraper:** By providing a "Place ID" (obtained from Google Developer Platform's Place ID Finder), Appify can scrape rich data about a specific location, including address, images, and popular times <a class="yt-timestamp" data-t="00:37:52">[00:37:52]</a>.
*   **Output:** Provides data in various file formats, including JSON, which is essential for the next step <a class="yt-timestamp" data-t="00:52:08">[00:52:08]</a>.

### 3. Claude / ChatGPT (Content Generation)
*   **Purpose:** To transform scraped data into structured, human-like content and generate SEO elements like schema markup <a class="yt-timestamp" data-t="00:39:40">[00:39:40]</a>. Claude is preferred for its more "human" output <a class="yt-timestamp" data-t="00:39:39">[00:39:39]</a>. This is a key step in [[Using AI for writing and content creation | AI content creation]] and [[Using AI in Content Creation | effective content creation]].
*   **Input:** The raw JSON data from Appify is fed into Claude <a class="yt-timestamp" data-t="00:40:09">[00:40:09]</a>.
*   **Prompt Engineering:**
    *   Users instruct Claude on the desired output format and specific information (e.g., "place name," "website URL with reference appended," "best time for date night," "scoring algorithm") <a class="yt-timestamp" data-t="00:53:56">[00:53:56]</a>.
    *   A key technique is to use asterisks (`**field name**:` followed by instructions) to format the output, allowing for easy parsing in later steps <a class="yt-timestamp" data-t="00:58:08">[00:58:08]</a>.
    *   This step is crucial for [[Using AI for content automation | content automation]].
    *   Claude can even generate algorithms (e.g., for scoring restaurants based on reviews) or create Chrome extensions for specific data scraping needs <a class="yt-timestamp" data-t="01:02:26">[01:02:26]</a>.
    *   Prompts can be refined to ensure desired language (e.g., UK English, Canadian French) and to avoid generic AI-sounding language <a class="yt-timestamp" data-t="01:19:00">[01:19:00]</a>.

### 4. Zapier (Automation & Integration)
*   **Purpose:** Connects all the tools to automate the entire content generation and publishing workflow <a class="yt-timestamp" data-t="00:59:04">[00:59:04]</a>.
*   **Workflow ("Zap"):**
    1.  **Trigger:** A new entry in a Google Sheet (e.g., a "Place ID") initiates the process <a class="yt-timestamp" data-t="00:45:03">[00:45:03]</a>.
    2.  **Appify Step:** Zapier calls the Appify Google Business Scraper using the Place ID to retrieve data <a class="yt-timestamp" data-t="00:47:02">[00:47:02]</a>.
    3.  **Code Step (Python):** A small Python script within Zapier extracts the full JSON data string from Appify's output, as Appify breaks down data into individual strings which Claude cannot process efficiently <a class="yt-timestamp" data-t="00:51:55">[00:51:55]</a>.
    4.  **Claude Step:** The extracted JSON string is sent to Claude with specific prompts to generate structured content <a class="yt-timestamp" data-t="00:53:35">[00:53:35]</a>.
    5.  **Code Step (Python) - Parsing Claude's Output:** Another Python script parses Claude's output (using the asterisk markers) to break it into individual, named data strings (e.g., `place_name`, `total_score`, `website_url`) <a class="yt-timestamp" data-t="01:07:38">[01:07:38]</a>. This ensures data is clean for Webflow.
    6.  **Webflow Step:** The parsed data is automatically sent to Webflow's CMS, populating the corresponding fields in the designated collection (e.g., "restaurants") and creating a new page for the listing <a class="yt-timestamp" data-t="01:08:52">[01:08:52]</a>.
*   **Scalability:** Allows for mass uploading of data from a single Google Sheet entry, automating the creation of thousands of pages <a class="yt-timestamp" data-t="00:45:31">[00:45:31]</a>.
