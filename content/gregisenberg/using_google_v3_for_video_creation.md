### Idea Generation and Scripting
The process begins with a "kernel of an idea hook" that is unexpected <a class="yt-timestamp" data-t="00:06:12">[00:06:12]</a> <a class="yt-timestamp" data-t="00:06:15">[00:06:15]</a>. For the pharmaceutical ad, the initial concept involved a placebo pill that leads to a puppy delivery <a class="yt-timestamp" data-t="00:07:02">[00:07:02]</a> <a class="yt-timestamp" data-t="00:07:06">[00:07:06]</a>. This idea was refined to a pill that secretes pheromones to attract puppies, adding a scientific breakthrough twist <a class="yt-timestamp" data-t="00:07:14">[00:07:14]</a> <a class="yt-timestamp" data-t="00:07:19">[00:07:19]</a>.

[[using_ai_for_content_creation | ChatGPT]] is used to generate initial, generalized scripts, which are then refined through iterative prompting <a class="yt-timestamp" data-t="00:07:37">[00:07:37]</a> <a class="yt-timestamp" data-t="00:07:46">[00:07:46]</a>. Users can ask [[using_ai_for_content_creation | ChatGPT]] to generate ideas for viral content or specific twists, such as an SNL-style satirical take on a commercial <a class="yt-timestamp" data-t="00:09:09">[00:09:09]</a> <a class="yt-timestamp" data-t="00:09:12">[00:09:12]</a>.

> "Okay, I know I can explore this type of commercial, but I want to have some sort of twist. Can you give me five different twists on a pharmaceutical commercial that could be really funny if it's, you know, like let's just say it's SNL style?" <a class="yt-timestamp" data-t="00:09:32">[00:09:32]</a>

To enhance humor, users can specify a comedian's style (e.g., "create jokes that Seth Rogan would like") <a class="yt-timestamp" data-t="00:15:44">[00:15:44]</a> <a class="yt-timestamp" data-t="00:15:46">[00:15:46]</a>. It's recommended to experiment with multiple LLMs like [[using_ai_for_content_creation | ChatGPT]], Claude, and Grok (especially its "unhinged" mode) for diverse outputs <a class="yt-timestamp" data-t="00:20:05">[00:20:05]</a> <a class="yt-timestamp" data-t="00:12:21">[00:12:21]</a>.

After initial script generation, the process involves continuously feeding favorite lines back into the LLM to refine and generate more "zingers" <a class="yt-timestamp" data-t="00:14:18">[00:14:18]</a> <a class="yt-timestamp" data-t="00:14:48">[00:14:48]</a>. Approximately 70% of the lines in the viral commercial originated from [[using_ai_for_content_creation | ChatGPT]] through this iterative process <a class="yt-timestamp" data-t="00:15:21">[00:15:21]</a> <a class="yt-timestamp" data-t="00:15:23">[00:15:23]</a>.

### Prompting for Google V3
Once the script is finalized, each dialogue line is converted into a prompt for Google V3 <a class="yt-timestamp" data-t="00:16:32">[00:16:32]</a> <a class="yt-timestamp" data-t="00:16:34">[00:16:34]</a>. The prompt structure is relatively simple, focusing on describing what's happening and the dialogue <a class="yt-timestamp" data-t="00:17:47">[00:17:47]</a> <a class="yt-timestamp" data-t="00:18:04">[00:18:04]</a>. Specific details like pose, shot type, camera type, and background elements can be included <a class="yt-timestamp" data-t="00:17:56">[00:17:56]</a> <a class="yt-timestamp" data-t="00:18:00">[00:18:00]</a>.

Each shot in V3 costs approximately $2 to $3 <a class="yt-timestamp" data-t="00:18:31">[00:18:31]</a> <a class="yt-timestamp" data-t="00:18:33">[00:18:33]</a>. It's recommended to run prompts one at a time to get feedback and adjust <a class="yt-timestamp" data-t="00:19:30">[00:19:30]</a> <a class="yt-timestamp" data-t="00:19:31">[00:19:31]</a>.

Limitations and workarounds for V3:
*   **Audio Issues**: V3's audio generation can be inconsistent, sometimes producing no audio or "whack" sounds <a class="yt-timestamp" data-t="00:18:40">[00:18:40]</a> <a class="yt-timestamp" data-t="00:18:41">[00:18:41]</a>. Avoiding quotes in prompts can reduce weird gibberish subtitles <a class="yt-timestamp" data-t="00:19:09">[00:19:09]</a> <a class="yt-timestamp" data-t="00:19:16">[00:19:16]</a>.
*   **Character Consistency**: V3 is currently text-to-video, meaning there's no inherent character consistency from shot to shot <a class="yt-timestamp" data-t="00:20:30">[00:20:30]</a> <a class="yt-timestamp" data-t="00:20:31">[00:20:31]</a>. To maintain a similar look, characters must be described in detail the same way for every prompt <a class="yt-timestamp" data-t="00:20:52">[00:20:52]</a> <a class="yt-timestamp" data-t="00:20:54">[00:20:54]</a>.
*   **Shot Iteration**: Expect a 3:1 to 5:1 ratio of generated clips to good ones <a class="yt-timestamp" data-t="00:23:44">[00:23:44]</a> <a class="yt-timestamp" data-t="00:23:52">[00:23:52]</a>.
