## The Smallville Simulation

The research populates a sandbox environment, reminiscent of the game "The Sims" or "Pokemon," with 25 generative agents <a class="yt-timestamp" data-t="01:37:00">[01:37:00]</a> <a class="yt-timestamp" data-t="01:39:00">[01:39:00]</a> <a class="yt-timestamp" data-t="01:45:00">[01:45:00]</a> <a class="yt-timestamp" data-t="01:47:00">[01:47:00]</a> <a class="yt-timestamp" data-t="01:55:00">[01:55:00]</a>. This environment, called "Smallville," features common affordances of a small village including houses, groceries, a pharmacy, a college, a bar, and a park, all structured as a tree data structure <a class="yt-timestamp" data-t="23:57:00">[23:57:00]</a> <a class="yt-timestamp" data-t="24:00:00">[24:00:00]</a> <a class="yt-timestamp" data-t="24:10:00">[24:10:00]</a> <a class="yt-timestamp" data-t="24:12:00">[24:12:00]</a> <a class="yt-timestamp" data-t="01:02:06">[01:02:06]</a> <a class="yt-timestamp" data-t="01:02:08">[01:02:08]</a>. Each agent is represented by a 2D "Sprite" avatar and given a one-paragraph natural language description of their identity and initial memories <a class="yt-timestamp" data-t="26:25:00">[26:25:00]</a> <a class="yt-timestamp" data-t="26:28:00">[26:28:00]</a> <a class="yt-timestamp" data-t="26:30:00">[26:30:00]</a> <a class="yt-timestamp" data-t="27:14:00">[27:14:00]</a> <a class="yt-timestamp" data-t="27:16:00">[27:16:00]</a>.

## Agent Architecture

The architecture of these generative agents is based on three main components:
*   **Memory Stream:** A long-term memory that records all of the agent's experiences in natural language <a class="yt-timestamp" data-t="08:06:00">[08:06:00]</a> <a class="yt-timestamp" data-t="08:07:00">[08:07:00]</a> <a class="yt-timestamp" data-t="08:09:00">[08:09:00]</a>. Each observation, such as "Maria is studying for a chemistry test while drinking coffee" or "the refrigerator is empty," is stored as text with a timestamp <a class="yt-timestamp" data-t="46:07:00">[46:07:00]</a> <a class="yt-timestamp" data-t="46:09:00">[46:09:00]</a> <a class="yt-timestamp" data-t="46:12:00">[46:12:00]</a> <a class="yt-timestamp" data-t="46:15:00">[46:15:00]</a> <a class="yt-timestamp" data-t="46:17:00">[46:17:00]</a> <a class="yt-timestamp" data-t="46:20:00">[46:20:00]</a>. A retrieval model combines recency, importance, and relevance to fetch pertinent memories <a class="yt-timestamp" data-t="08:14:00">[08:14:00]</a> <a class="yt-timestamp" data-t="08:16:00">[08:16:00]</a>.
    *   **Recency:** An exponential decay function over the number of sandbox game hours since the memory was last retrieved <a class="yt-timestamp" data-t="48:19:00">[48:19:00]</a> <a class="yt-timestamp" data-t="48:22:00">[48:22:00]</a>.
    *   **Importance:** Distinguished by assigning a score (1-10) to memories based on how "poignant" the agent believes them to be, e.g., a breakup scores high, breakfast low <a class="yt-timestamp" data-t="49:00:00">[49:00:00]</a> <a class="yt-timestamp" data-t="49:03:00">[49:03:00]</a> <a class="yt-timestamp" data-t="49:05:00">[49:05:00]</a> <a class="yt-timestamp" data-t="49:10:00">[49:10:00]</a> <a class="yt-timestamp" data-t="49:11:00">[49:11:00]</a> <a class="yt-timestamp" data-t="49:14:00">[49:14:00]</a> <a class="yt-timestamp" data-t="49:16:00">[49:16:00]</a> <a class="yt-timestamp" data-t="49:39:00">[49:39:00]</a> <a class="yt-timestamp" data-t="49:41:00">[49:41:00]</a> <a class="yt-timestamp" data-t="49:42:00">[49:42:00]</a> <a class="yt-timestamp" data-t="49:45:00">[49:45:00]</a>.
    *   **Relevance:** Calculated using cosine similarity between the embedding vectors of the memory and the current query, meaning how related a memory is to the current situation <a class="yt-timestamp" data-t="50:01:00">[50:01:00]</a> <a class="yt-timestamp" data-t="50:02:00">[50:02:00]</a> <a class="yt-timestamp" data-t="50:04:00">[50:04:00]</a> <a class="yt-timestamp" data-t="50:07:00">[50:07:00]</a> <a class="yt-timestamp" data-t="50:09:00">[50:09:00]</a> <a class="yt-timestamp" data-t="50:11:00">[50:11:00]</a> <a class="yt-timestamp" data-t="50:13:00">[50:13:00]</a> <a class="yt-timestamp" data-t="50:15:00">[50:15:00]</a> <a class="yt-timestamp" data-t="50:16:00">[50:16:00]</a> <a class="yt-timestamp" data-t="50:26:00">[50:26:00]</a> <a class="yt-timestamp" data-t="50:27:00">[50:27:00]</a> <a class="yt-timestamp" data-t="50:29:00">[50:29:00]</a> <a class="yt-timestamp" data-t="50:31:00">[50:31:00]</a> <a class="yt-timestamp" data-t="50:32:00">[50:32:00]</a> <a class="yt-timestamp" data-t="50:33:00">[50:33:00]</a>. This leverages [[generative_latent_spaces_in_ai | generative latent spaces in AI]] for retrieval.
*   **Reflection:** Synthesizes memories into higher-level inferences over time, generating abstract thoughts periodically (roughly two to three times a day) <a class="yt-timestamp" data-t="08:26:00">[08:26:00]</a> <a class="yt-timestamp" data-t="08:28:00">[08:28:00]</a> <a class="yt-timestamp" data-t="08:29:00">[08:29:00]</a> <a class="yt-timestamp" data-t="52:50:00">[52:50:00]</a> <a class="yt-timestamp" data-t="52:52:00">[52:52:00]</a> <a class="yt-timestamp" data-t="52:56:00">[52:56:00]</a> <a class="yt-timestamp" data-t="53:01:00">[53:01:00]</a> <a class="yt-timestamp" data-t="53:04:00">[53:04:00]</a>. *   **Planning:** Allows agents to plan actions over a longer time horizon to ensure coherent and believable behavior <a class="yt-timestamp" data-t="54:23:00">[54:23:00]</a> <a class="yt-timestamp" data-t="54:25:00">[54:25:00]</a> <a class="yt-timestamp" data-t="54:26:00">[54:26:00]</a> <a class="yt-timestamp" data-t="54:28:00">[54:28:00]</a> <a class="yt-timestamp" data-t="54:30:00">[54:30:00]</a> <a class="yt-timestamp" data-t="54:31:00">[54:31:00]</a>. The planning is top-down, generating broad strokes for the day and then recursively adding more detail, which can be regenerated based on new observations <a class="yt-timestamp" data-t="56:39:00">[56:39:00]</a> <a class="yt-timestamp" data-t="56:41:00">[56:41:00]</a> <a class="yt-timestamp" data-t="56:43:00">[56:43:00]</a> <a class="yt-timestamp" data-t="58:49:00">[58:49:00]</a> <a class="yt-timestamp" data-t="58:51:00">[58:51:00]</a> <a class="yt-timestamp" data-t="58:53:00">[58:53:00]</a>.

The entire system functions on an "Observation, Planning, and Reflection" (or Sense Plan Act) paradigm, akin to robotics, where perceptions are continuously stored, relevant memories retrieved, and actions determined based on these processes <a class="yt-timestamp" data-t="04:54:00">[04:54:00]</a> <a class="yt-timestamp" data-t="05:07:00">[05:07:00]</a> <a class="yt-timestamp" data-t="05:56:00">[05:56:00]</a> <a class="yt-timestamp" data-t="05:57:00">[05:57:00]</a> <a class="yt-timestamp" data-t="05:59:00">[05:59:00]</a> <a class="yt-timestamp" data-t="01:53:00">[01:53:00]</a> <a class="yt-timestamp" data-t="01:56:00">[01:56:00]</a>. Each agent operates with its own LLM, specifically GPT-3.5 Turbo, driving these cognitive functions <a class="yt-timestamp" data-t="09:09:00">[09:09:00]</a> <a class="yt-timestamp" data-t="09:12:00">[09:12:00]</a> <a class="yt-timestamp" data-t="09:43:00">[09:43:00]</a> <a class="yt-timestamp" data-t="09:46:00">[09:46:00]</a> <a class="yt-timestamp" data-t="09:47:00">[09:47:00]</a> <a class="yt-timestamp" data-t="01:00:09">[01:00:09]</a>.
