Logarithms are fundamentally about figuring out what power you have to raise a base number to, to get another number <a class="yt-timestamp" data-t="01:00:00">[01:00:00]</a>.

## [[relationship_between_exponents_and_logarithms | Relationship with Exponents]]

To understand logarithms, it's helpful to first recall how exponents work <a class="yt-timestamp" data-t="00:06:00">[00:06:00]</a>. For example, 2 to the fourth power means 2 multiplied by itself 4 times <a class="yt-timestamp" data-t="00:09:00">[00:09:00]</a>. This calculation yields:
2 × 2 × 2 × 2 = 16 <a class="yt-timestamp" data-t="00:13:00">[00:13:00]</a>.
So, 2 to the fourth power is 16 <a class="yt-timestamp" data-t="00:26:00">[00:26:00]</a>.

Logarithms pose the inverse question <a class="yt-timestamp" data-t="00:32:00">[00:32:00]</a>: If you start with 2 and want to reach 16 by raising 2 to some power (let's call it 'x'), what does 'x' have to be? <a class="yt-timestamp" data-t="00:42:00">[00:42:00]</a> In this case, 'x' would be 4 <a class="yt-timestamp" data-t="00:52:00">[00:52:00]</a>.

## [[logarithm_notation | Logarithm Notation]]

The way to [[logarithm_notation | denote this with logarithm notation]] is by writing:
log base 2 of 16 = x <a class="yt-timestamp" data-t="01:23:00">[01:23:00]</a>

This logarithmic statement is completely equivalent to the exponential statement:
2<sup>x</sup> = 16 <a class="yt-timestamp" data-t="01:39:00">[01:39:00]</a>

Both statements ask the same question: "What power do I need to raise 2 to, to get 16?" <a class="yt-timestamp" data-t="01:49:00">[01:49:00]</a> The answer, in both cases, is x = 4 <a class="yt-timestamp" data-t="01:57:00">[01:57:00]</a>.

## [[evaluating_logarithmic_expressions | Evaluating Logarithmic Expressions]]

Let's look at more [[examples_of_logarithmic_calculations | examples of logarithmic calculations]] to practice [[evaluating_logarithmic_expressions | evaluating logarithmic expressions]] <a class="yt-timestamp" data-t="02:01:00">[02:01:00]</a>.

### Example 1: log<sub>3</sub>(81)

Consider the expression:
log<sub>3</sub>(81) <a class="yt-timestamp" data-t="02:12:00">[02:12:00]</a>

This asks: "What power do we have to raise 3 to, to get to 81?" <a class="yt-timestamp" data-t="02:25:00">[02:25:00]</a>
We can set this equal to 'x' and rephrase it as:
3<sup>x</sup> = 81 <a class="yt-timestamp" data-t="02:33:00">[02:33:00]</a>

By experimenting with powers of 3:
*   3<sup>1</sup> = 3 <a class="yt-timestamp" data-t="03:21:00">[03:21:00]</a>
*   3<sup>2</sup> = 9 <a class="yt-timestamp" data-t="03:25:00">[03:25:00]</a>
*   3<sup>3</sup> = 27 <a class="yt-timestamp" data-t="03:25:00">[03:25:00]</a>
*   3<sup>4</sup> = 81 <a class="yt-timestamp" data-t="03:34:00">[03:34:00]</a>

Therefore, log<sub>3</sub>(81) = 4 <a class="yt-timestamp" data-t="03:41:00">[03:41:00]</a>.

### Example 2: log<sub>6</sub>(216)

Evaluate:
log<sub>6</sub>(216) <a class="yt-timestamp" data-t="04:09:00">[04:09:00]</a>

This asks: "What power do we have to raise 6 to, to get 216?" <a class="yt-timestamp" data-t="04:17:00">[04:17:00]</a>

*   6<sup>1</sup> = 6 <a class="yt-timestamp" data-t="04:22:00">[04:22:00]</a>
*   6<sup>2</sup> = 36 <a class="yt-timestamp" data-t="04:22:00">[04:22:00]</a>
*   6<sup>3</sup> = 36 × 6 = 216 <a class="yt-timestamp" data-t="04:27:00">[04:27:00]</a>

So, log<sub>6</sub>(216) = 3 <a class="yt-timestamp" data-t="04:44:00">[04:44:00]</a>.

### Example 3: log<sub>2</sub>(64)

Evaluate:
log<sub>2</sub>(64) <a class="yt-timestamp" data-t="04:56:00">[04:56:00]</a>

This asks: "What exponent do I have to raise the base 2 to, to get to 64?" <a class="yt-timestamp" data-t="05:06:00">[05:06:00]</a>

*   2<sup>1</sup> = 2 <a class="yt-timestamp" data-t="05:18:00">[05:18:00]</a>
*   2<sup>2</sup> = 4 <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
*   2<sup>3</sup> = 8 <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
*   2<sup>4</sup> = 16 <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
*   2<sup>5</sup> = 32 <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
*   2<sup>6</sup> = 64 <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>

Thus, log<sub>2</sub>(64) = 6 <a class="yt-timestamp" data-t="05:34:00">[05:34:00]</a>.

## [[special_cases_in_logarithms | Special Cases in Logarithms]]

### Logarithm of 1

Consider:
log<sub>100</sub>(1) <a class="yt-timestamp" data-t="05:51:00">[05:51:00]</a>

This asks: "What power do I have to raise 100 to, to get to 1?" <a class="yt-timestamp" data-t="06:10:00">[06:10:00]</a>
If we set this to 'x':
100<sup>x</sup> = 1 <a class="yt-timestamp" data-t="06:24:00">[06:24:00]</a>

Any non-zero number raised to the power of 0 is equal to 1 <a class="yt-timestamp" data-t="06:31:00">[06:31:00]</a>.
Therefore, x = 0 <a class="yt-timestamp" data-t="06:34:00">[06:34:00]</a>.

This leads to a general rule: log base anything (except 0) of 1 is always equal to 0 <a class="yt-timestamp" data-t="06:44:00">[06:44:00]</a>.
log<sub>b</sub>(1) = 0 (where b ≠ 0) <a class="yt-timestamp" data-t="06:44:00">[06:44:00]</a>
