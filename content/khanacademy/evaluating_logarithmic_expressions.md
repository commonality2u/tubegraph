Evaluating a [[calculating_logarithms_with_examples | logarithmic expression]] fundamentally involves determining what power a base number must be raised to in order to obtain another specific number <a class="yt-timestamp" data-t="01:00:00">[01:00:00]</a>. This is the inverse operation of [[understanding_exponents | exponents]] <a class="yt-timestamp" data-t="00:32:00">[00:32:00]</a>.

## Understanding the Concept

If you know that 2 raised to the fourth power equals 16 (2 x 2 x 2 x 2 = 16) <a class="yt-timestamp" data-t="00:13:18">[00:13:18]</a>, then evaluating a logarithm asks the reverse: "To what power must 2 be raised to get 16?" <a class="yt-timestamp" data-t="00:35:00">[00:35:00]</a>. The answer, in this case, is 4 <a class="yt-timestamp" data-t="00:52:00">[00:52:00]</a>.

## Logarithm Notation

Logarithms are denoted using "log" followed by a base number as a subscript, and then the number being evaluated.

For example, the question "What power do I need to raise 2 to, to get 16?" is written as:
`log`<sub>2</sub> `(16)` <a class="yt-timestamp" data-t="01:23:00">[01:23:00]</a>

If `log`<sub>2</sub> `(16) = x`, this statement is equivalent to the exponential equation `2`<sup>`x`</sup> `= 16` <a class="yt-timestamp" data-t="01:39:00">[01:39:00]</a>. In this example, `x` would be 4 <a class="yt-timestamp" data-t="01:57:00">[01:57:00]</a>.

## Examples of Evaluating Logarithmic Expressions

### Example 1: `log`<sub>3</sub> `(81)`

To evaluate `log`<sub>3</sub> `(81)` <a class="yt-timestamp" data-t="02:12:00">[02:12:00]</a>, we ask: "What power do we have to raise 3 to, to get 81?" <a class="yt-timestamp" data-t="02:25:00">[02:25:00]</a>.
*   3<sup>1</sup> = 3 <a class="yt-timestamp" data-t="03:21:00">[03:21:00]</a>
*   3<sup>2</sup> = 9 <a class="yt-timestamp" data-t="03:23:00">[03:23:00]</a>
*   3<sup>3</sup> = 27 <a class="yt-timestamp" data-t="03:25:00">[03:25:00]</a>
*   3<sup>4</sup> = 81 <a class="yt-timestamp" data-t="03:27:00">[03:27:00]</a>
Therefore, `log`<sub>3</sub> `(81) = 4` <a class="yt-timestamp" data-t="03:36:00">[03:36:00]</a>.

### Example 2: `log`<sub>6</sub> `(216)`

To evaluate `log`<sub>6</sub> `(216)` <a class="yt-timestamp" data-t="04:09:00">[04:09:00]</a>, we ask: "What power do we have to raise 6 to, to get 216?" <a class="yt-timestamp" data-t="04:17:00">[04:17:00]</a>.
*   6<sup>1</sup> = 6 <a class="yt-timestamp" data-t="04:22:00">[04:22:00]</a>
*   6<sup>2</sup> = 36 <a class="yt-timestamp" data-t="04:24:00">[04:24:00]</a>
*   6<sup>3</sup> = 216 <a class="yt-timestamp" data-t="04:27:00">[04:27:00]</a>
Therefore, `log`<sub>6</sub> `(216) = 3` <a class="yt-timestamp" data-t="04:34:00">[04:34:00]</a>.

### Example 3: `log`<sub>2</sub> `(64)`

To evaluate `log`<sub>2</sub> `(64)` <a class="yt-timestamp" data-t="04:56:00">[04:56:00]</a>, we ask: "What power do we have to raise 2 to, to get 64?" <a class="yt-timestamp" data-t="05:06:00">[05:06:00]</a>.
*   2<sup>1</sup> = 2 <a class="yt-timestamp" data-t="05:18:00">[05:18:00]</a>
*   2<sup>2</sup> = 4 <a class="yt-timestamp" data-t="05:20:00">[05:20:00]</a>
*   2<sup>3</sup> = 8 <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
*   2<sup>4</sup> = 16 <a class="yt-timestamp" data-t="05:23:00">[05:23:00]</a>
*   2<sup>5</sup> = 32 <a class="yt-timestamp" data-t="05:24:00">[05:24:00]</a>
*   2<sup>6</sup> = 64 <a class="yt-timestamp" data-t="05:25:00">[05:25:00]</a>
Therefore, `log`<sub>2</sub> `(64) = 6` <a class="yt-timestamp" data-t="05:34:00">[05:34:00]</a>.

### Example 4: `log`<sub>100</sub> `(1)`

To evaluate `log`<sub>100</sub> `(1)` <a class="yt-timestamp" data-t="05:51:00">[05:51:00]</a>, we ask: "What power do we have to raise 100 to, to get 1?" <a class="yt-timestamp" data-t="06:15:00">[06:15:00]</a>.
Any non-zero number raised to the power of 0 is equal to 1 <a class="yt-timestamp" data-t="06:31:00">[06:31:00]</a>.
Therefore, `100`<sup>`0`</sup> `= 1` <a class="yt-timestamp" data-t="06:36:00">[06:36:00]</a>.
Thus, `log`<sub>100</sub> `(1) = 0` <a class="yt-timestamp" data-t="06:36:00">[06:36:00]</a>.

***

> [!NOTE] Logarithm of 1
> Log base any number (that is not 0) of 1 will always be equal to 0 <a class="yt-timestamp" data-t="06:44:00">[06:44:00]</a>. This is due to the [[properties_of_logarithms | property]] that any non-zero number raised to the power of 0 equals 1 <a class="yt-timestamp" data-t="06:50:00">[06:50:00]</a>.
