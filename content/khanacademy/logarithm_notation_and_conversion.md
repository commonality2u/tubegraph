Logarithms are a mathematical concept fundamentally about figuring out what power you have to raise a base number to, to get another number <a class="yt-timestamp" data-t="01:00:00">[01:00:00]</a>. This concept is closely related to [[understanding_exponents | exponents]] and provides an [[inverse_relationship_between_logarithms_and_exponents | inverse perspective]] to them.

## Understanding Exponents

Before diving into logarithms, it's helpful to recall [[understanding_exponents | exponents]]. For example, `2` to the fourth power means `2` multiplied by itself `4` times <a class="yt-timestamp" data-t="00:00:09">[00:00:09]</a>.
*   `2 × 2 × 2 × 2 = 16` <a class="yt-timestamp" data-t="00:00:13">[00:00:13]</a>.
*   So, `2^4 = 16` <a class="yt-timestamp" data-t="00:00:34">[00:00:34]</a>.

Logarithms pose this question in reverse: If we know `2` raised to *some power* equals `16`, what is that power? <a class="yt-timestamp" data-t="00:00:35">[00:00:35]</a>. In this case, the power (`x`) would be `4` <a class="yt-timestamp" data-t="00:00:52">[00:00:52]</a>.

## Logarithm Notation

The way to denote this concept using logarithm notation is `log` base `b` of `x` equals `y` <a class="yt-timestamp" data-t="01:08:08">[01:08:08]</a>.

For the example above (`2` to what power equals `16`?):
*   We would write: `log` base `2` of `16` is equal to `x` <a class="yt-timestamp" data-t="01:18:18">[01:18:18]</a>.
*   This statement (`log₂16 = x`) and the exponential statement (`2^x = 16`) are completely equivalent <a class="yt-timestamp" data-t="01:39:00">[01:39:00]</a>.
*   Both are asking: "What power do I need to raise `2` to, to get `16`?" <a class="yt-timestamp" data-t="01:49:00">[01:49:00]</a>.
*   The answer is `x = 4` <a class="yt-timestamp" data-t="01:59:00">[01:59:00]</a>.

> [!NOTE]
> The base number (e.g., `2` in `log₂16`) is written as a small subscript <a class="yt-timestamp" data-t="05:11:00">[05:11:00]</a>.

## Converting and [[evaluating_logarithmic_expressions | Evaluating Logarithmic Expressions]]

To [[evaluating_logarithmic_expressions | evaluate a logarithmic expression]], you are essentially determining the exponent. You can set the expression equal to a variable (e.g., `x`) and then convert it into its equivalent exponential form <a class="yt-timestamp" data-t="02:33:00">[02:33:00]</a>.

Here are several [[calculating_logarithms_with_examples | examples]]:

### Example 1: `log₃81`
*   **Question**: What power do we have to raise `3` to, to get `81`? <a class="yt-timestamp" data-t="02:25:00">[02:25:00]</a>
*   **Conversion**: Set `log₃81 = x`, which is equivalent to `3^x = 81` <a class="yt-timestamp" data-t="02:33:00">[02:33:00]</a>.
*   **Calculation**:
    *   `3^1 = 3` <a class="yt-timestamp" data-t="03:21:00">[03:21:00]</a>
    *   `3^2 = 9` <a class="yt-timestamp" data-t="03:25:00">[03:25:00]</a>
    *   `3^3 = 27` <a class="yt-timestamp" data-t="03:25:00">[03:25:00]</a>
    *   `3^4 = 27 × 3 = 81` <a class="yt-timestamp" data-t="03:25:00">[03:25:00]</a>
*   **Result**: `log₃81 = 4` <a class="yt-timestamp" data-t="03:36:00">[03:36:00]</a>

### Example 2: `log₆216`
*   **Question**: What power do we have to raise `6` to, to get `216`? <a class="yt-timestamp" data-t="04:17:00">[04:17:00]</a>
*   **Conversion**: Equivalent to `6^x = 216`.
*   **Calculation**:
    *   `6^1 = 6` <a class="yt-timestamp" data-t="04:22:00">[04:22:00]</a>
    *   `6^2 = 36` <a class="yt-timestamp" data-t="04:22:00">[04:22:00]</a>
    *   `6^3 = 36 × 6 = 216` <a class="yt-timestamp" data-t="04:27:00">[04:27:00]</a>
*   **Result**: `log₆216 = 3` <a class="yt-timestamp" data-t="04:44:00">[04:44:00]</a>

### Example 3: `log₂64`
*   **Question**: What power do we have to raise `2` to, to get `64`? <a class="yt-timestamp" data-t="05:06:00">[05:06:00]</a>
*   **Conversion**: Equivalent to `2^x = 64`.
*   **Calculation**:
    *   `2^1 = 2` <a class="yt-timestamp" data-t="05:18:00">[05:18:00]</a>
    *   `2^2 = 4` <a class="yt-timestamp" data-t="05:18:00">[05:18:00]</a>
    *   `2^3 = 8` <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
    *   `2^4 = 16` <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
    *   `2^5 = 32` <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
    *   `2^6 = 64` <a class="yt-timestamp" data-t="05:22:00">[05:22:00]</a>
*   **Result**: `log₂64 = 6` <a class="yt-timestamp" data-t="05:38:00">[05:38:00]</a>

## [[undefined_logarithms_and_special_cases | Special Cases]]

### Logarithm of 1
Consider `log₁₀₀1` <a class="yt-timestamp" data-t="05:51:00">[05:51:00]</a>.
*   **Question**: What power do I have to raise `100` to, to get `1`? <a class="yt-timestamp" data-t="06:10:00">[06:10:00]</a>
*   **Conversion**: Set `log₁₀₀1 = x`, which is `100^x = 1` <a class="yt-timestamp" data-t="06:24:00">[06:24:00]</a>.
*   **Rule**: Any non-zero number raised to the `0` power is equal to `1` <a class="yt-timestamp" data-t="06:31:00">[06:31:00]</a>.
*   **Result**: `x = 0`, so `log₁₀₀1 = 0` <a class="yt-timestamp" data-t="06:36:00">[06:36:00]</a>.

In general, `log` base *anything* (not zero) of `1` is always equal to `0` <a class="yt-timestamp" data-t="06:44:00">[06:44:00]</a>.
