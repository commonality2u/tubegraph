## Easy Start Languages

These languages are designed to make [[learning_to_code_as_a_skill | programming]] as easy as possible for those with no prior experience <a class="yt-timestamp" data-t="00:00:51">[00:00:51]</a>.

*   **Scratch**
    *   Developed at MIT <a class="yt-timestamp" data-t="00:00:57">[00:00:57]</a>.
    *   Uses a drag-and-drop block interface, similar to Lego bricks, to represent variables, control flow, and operators <a class="yt-timestamp" data-t="00:01:02">[00:01:02]</a>.
    *   Makes the thinking process behind programming much more accessible <a class="yt-timestamp" data-t="00:01:10">[00:01:10]</a>.
    *   Often considered the best place to start for absolute beginners <a class="yt-timestamp" data-t="00:00:56">[00:00:56]</a>.
*   **BASIC (Beginner's All-purpose Symbolic Instruction Code)**
    *   Created at Dartmouth in 1964 <a class="yt-timestamp" data-t="00:01:20">[00:01:20]</a>.
    *   Provides fundamental commands like `print`, `go to`, and `for` <a class="yt-timestamp" data-t="00:01:27">[00:01:27]</a>.
    *   Was a primary choice for [[benefits_of_learning_programming_at_a_young_age | people learning to code]] for 50 years, often included in personal computers <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a>.

## Popular Dynamic High-Level Languages

This tier includes extremely popular dynamic high-level languages that are common starting points today <a class="yt-timestamp" data-t="00:01:37">[00:01:37]</a>. After learning one of these, you can build almost anything and pursue a career in software engineering without needing to delve deeper <a class="yt-timestamp" data-t="00:02:07">[00:02:07]</a>.

*   **Python**
    *   Most commonly recommended language for beginners today <a class="yt-timestamp" data-t="00:01:42">[00:01:42]</a>.
    *   Known for its minimal syntax, using indentation instead of curly braces or semicolons <a class="yt-timestamp" data-t="00:01:44">[00:01:44]</a>.
*   **JavaScript**
    *   Syntactically complex but essential for [[popular_programming_languages_for_web_development | web development]] <a class="yt-timestamp" data-t="00:01:54">[00:01:54]</a>.
    *   Almost every developer will encounter it at some point <a class="yt-timestamp" data-t="00:01:59">[00:01:59]</a>.

## Specialized Popular Languages

This level features languages that are widely used but cater to more specific programming domains <a class="yt-timestamp" data-t="00:02:18">[00:02:18]</a>.

*   **Scripting and Utility Languages**
    *   Used for command-line tasks and automating repetitive computer interactions <a class="yt-timestamp" data-t="00:02:22">[00:02:22]</a>.
    *   Examples include **Bash** and **PowerShell** <a class="yt-timestamp" data-t="00:02:25">[00:02:25]</a>.
*   **Web Development Languages**
    *   **HTML** and **CSS**: Used to define the structure and style of websites <a class="yt-timestamp" data-t="00:02:35">[00:02:35]</a>. When combined, they arguably form a Turing-complete language <a class="yt-timestamp" data-t="00:02:39">[00:02:39]</a>.
*   **Database Languages**
    *   **SQL (Structured Query Language)**: The most common language for interacting with databases <a class="yt-timestamp" data-t="00:02:53">[00:02:53]</a>. It's Turing-complete but used for reading and writing data in relational databases, not general programming <a class="yt-timestamp" data-t="00:02:58">[00:02:58]</a>.
*   **Other Dynamic Languages**
    *   **PHP**: Popular for building server-side web applications, especially since the 1990s <a class="yt-timestamp" data-t="00:03:09">[00:03:09]</a>.
    *   **Lua**: Easier and faster than Python, embedded in engines like Roblox and World of Warcraft <a class="yt-timestamp" data-t="00:03:16">[00:03:16]</a>.
    *   **Ruby**: An easy-to-learn object-oriented language often used for web applications with the Rails framework <a class="yt-timestamp" data-t="00:03:21">[00:03:21]</a>.
    *   **R**: Encountered in data science for statistics and data visualization <a class="yt-timestamp" data-t="00:03:27">[00:03:27]</a>.
    *   **Julia**: A modern option for scientific computing <a class="yt-timestamp" data-t="00:03:31">[00:03:31]</a>.

## Static Type Systems

As software complexity grows, a more rigid framework is often needed, which can be achieved with a static type system <a class="yt-timestamp" data-t="00:03:39">[00:03:39]</a>. This tier makes up a significant portion of production code worldwide <a class="yt-timestamp" data-t="00:03:47">[00:03:47]</a>.

*   **Java**
    *   Revolutionized programming with the Java Virtual Machine (JVM), allowing bytecode to run on any computer architecture from a single codebase <a class="yt-timestamp" data-t="00:03:50">[00:03:50]</a>.
    *   Has a complex syntax for beginners, but explicit types make code easier to understand and refactor <a class="yt-timestamp" data-t="00:04:01">[00:04:01]</a>.
*   **C# (C-sharp)**
    *   Developed by Microsoft, similar to Java <a class="yt-timestamp" data-t="00:04:16">[00:04:16]</a>.
    *   Used for game development with Unity, as well as web and desktop applications with the .NET framework <a class="yt-timestamp" data-t="00:04:21">[00:04:21]</a>.
*   **TypeScript**
    *   Microsoft's addition of a type system on top of JavaScript <a class="yt-timestamp" data-t="00:04:28">[00:04:28]</a>.
    *   Makes JavaScript easier to work with on large, complex projects <a class="yt-timestamp" data-t="00:04:32">[00:04:32]</a>.
*   **Mobile Development Languages**
    *   **Kotlin** for Android <a class="yt-timestamp" data-t="00:04:37">[00:04:37]</a>.
    *   **Swift** for iOS <a class="yt-timestamp" data-t="00:04:39">[00:04:39]</a>.
    *   **Dart** with the Flutter framework <a class="yt-timestamp" data-t="00:04:40">[00:04:40]</a>.
    *   These are statically typed, offering modern and concise ways with features like type inference to minimize boilerplate code <a class="yt-timestamp" data-t="00:04:42">[00:04:42]</a>.
*   **Go**
    *   A high-performance language developed at Google for [[lowlevel_systems_programming_languages | low-level systems]] <a class="yt-timestamp" data-t="00:04:51">[00:04:51]</a>.
    *   Designed as a replacement for C, with one of C's original creators, Ken Thompson, contributing to its design <a class="yt-timestamp" data-t="00:04:55">[00:04:55]</a>.
    *   Features concise syntax making it approachable for beginners <a class="yt-timestamp" data-t="00:05:00">[00:05:00]</a>.
    *   Includes a garbage collector, eliminating the need for manual memory management (unlike C) <a class="yt-timestamp" data-t="00:05:04">[00:05:04]</a>.

## Functional Languages

This level involves languages that developers explore when seeking alternative programming paradigms, often moving away from heavy object-oriented approaches <a class="yt-timestamp" data-t="00:05:16">[00:05:16]</a>. The core abstraction is the function, with immutable variables and functions having no side effects <a class="yt-timestamp" data-t="00:05:30">[00:05:30]</a>.

*   **Haskell**: The most famous purely functional language <a class="yt-timestamp" data-t="00:05:24">[00:05:24]</a>.
*   **F# (F-sharp)**: Microsoft's functional sister language to C#, also supports imperative and object-oriented programming, making it more approachable <a class="yt-timestamp" data-t="00:05:55">[00:05:55]</a>.
*   **Scala**: A good Java alternative, supporting both object-oriented and functional programming, runs on the JVM <a class="yt-timestamp" data-t="00:06:08">[00:06:08]</a>.
*   **Clojure**: Runs on the JVM, both functional and dynamic, suited for quick development but with a trade-off in type safety <a class="yt-timestamp" data-t="00:06:16">[00:06:16]</a>.
*   **OCaml**: Used extensively at Facebook <a class="yt-timestamp" data-t="00:06:26">[00:06:26]</a>.
*   **Elixir**: Features a Ruby-like syntax, capable of building high-performance real-time web applications <a class="yt-timestamp" data-t="00:06:29">[00:06:29]</a>.
*   **Elm**: A purely functional language that compiles to JavaScript, used for front-end UIs with zero runtime errors <a class="yt-timestamp" data-t="00:06:36">[00:06:36]</a>.

## Low-Level Systems Languages

These languages are [[lowlevel_systems_programming_languages | low-level systems languages]] that allow manual memory management and optimization. They are used to build operating system kernels and compilers that enable other programming languages <a class="yt-timestamp" data-t="00:06:48">[00:06:48]</a>.

*   **C**
    *   Used to build the Windows, Mac, and Linux operating system kernels <a class="yt-timestamp" data-t="00:06:59">[00:06:59]</a>.
    *   Its curly brace syntax influenced many other languages <a class="yt-timestamp" data-t="00:07:04">[00:07:04]</a>.
    *   Surprisingly, not that hard to learn, with a relatively small set of keywords <a class="yt-timestamp" data-t="00:07:07">[00:07:07]</a>.
    *   Effective use requires extensive knowledge of algorithms and computer architecture (e.g., coding data structures like hash maps manually) <a class="yt-timestamp" data-t="00:07:12">[00:07:12]</a>.
    *   Only supported procedural programming upon its release in 1969 <a class="yt-timestamp" data-t="00:07:22">[00:07:22]</a>.
*   **C++ (C-plus-plus)**
    *   Originally a superset of C, designed to extend it with object-oriented programming patterns like classes and inheritance <a class="yt-timestamp" data-t="00:07:29">[00:07:29]</a>.
    *   Extremely difficult to learn, known for manual memory management with pointers <a class="yt-timestamp" data-t="00:07:37">[00:07:37]</a>.
    *   Highly prolific for building optimized software like game engines and compilers <a class="yt-timestamp" data-t="00:07:51">[00:07:51]</a>.
*   **Rust**
    *   A modern choice for [[learning_lowerlevel_programming_languages_like_c | low-level programming]] <a class="yt-timestamp" data-t="00:07:58">[00:07:58]</a>.
    *   Does not have a garbage collector, but uses "borrow checking" instead of pointers for memory management <a class="yt-timestamp" data-t="00:08:03">[00:08:03]</a>.
    *   Makes it easier to write memory-safe programs and consistently ranks as a most-loved language <a class="yt-timestamp" data-t="00:08:11">[00:08:11]</a>.

## Modern Unconventional Languages

This tier introduces modern languages that are less commonly known but offer unique approaches to programming.

*   **V**: A high-performance systems language similar to Go, but uses "auto-free" innovation for memory safety without a garbage collector or borrow checking <a class="yt-timestamp" data-t="00:08:24">[00:08:24]</a>.
*   **Zig**: Designed to simplify [[getting_started_with_c_programming | low-level programming]] by eliminating macros and meta-programming, explicit about memory management, and can cross-compile C and C++ <a class="yt-timestamp" data-t="00:08:43">[00:08:43]</a>.
*   **Nim**: High-performance, expressive like Python, but statically typed. Features a tunable garbage collector that can be turned off for manual memory management <a class="yt-timestamp" data-t="00:08:57">[00:08:57]</a>.
*   **Carbon**: Announced by Google as a successor to C++, designed to fully interoperate with legacy C++ codebases <a class="yt-timestamp" data-t="00:09:09">[00:09:09]</a>.
*   **Solidity**: A statically typed, object-oriented language designed for implementing smart contracts, especially on the Ethereum blockchain <a class="yt-timestamp" data-t="00:09:17">[00:09:17]</a>.
*   **Hack**: From Facebook, designed to interoperate with PHP, offering better performance and a type system for scaling large websites <a class="yt-timestamp" data-t="00:09:26">[00:09:26]</a>.

## Historically Important and Niche Languages

This level features languages that are still widely used or historically significant, though a beginner might not choose them for initial programming <a class="yt-timestamp" data-t="00:09:45">[00:09:45]</a>.

*   **Fortran**: The first high-level programming language and most popular for many years until C emerged <a class="yt-timestamp" data-t="00:09:51">[00:09:51]</a>.
*   **Lisp**: Invented in 1958, pioneered dynamic typing, higher-order functions, recursion, and REPLs, influencing languages like JavaScript <a class="yt-timestamp" data-t="00:09:59">[00:10:00]</a>.
*   **Algol**: A big, complex language from 1960 that influenced C and C++ with its type system and use of expressions <a class="yt-timestamp" data-t="00:10:15">[00:10:15]</a>.
*   **COBOL**: Born in 1959, still used today by over 40% of banking systems, with over 200 billion lines of code in production <a class="yt-timestamp" data-t="00:10:28">[00:10:28]</a>.
*   **APL (A Programming Language)**: Appeared in 1962, implements linear algebra directly, using a multi-dimensional array as a central data type, leading to terse code resembling mathematical notation <a class="yt-timestamp" data-t="00:10:39">[00:10:39]</a>.
*   **Pascal**: Invented in 1970, a procedural language with a familiar syntax and fast compile times, becoming the most popular language in the early 1980s before C <a class="yt-timestamp" data-t="00:10:54">[00:10:54]</a>.
*   **Simula**: The first object-oriented language, inspiring Smalltalk, which in turn inspired Python, Java, and Ruby <a class="yt-timestamp" data-t="00:11:10">[00:11:10]</a>.
*   **Erlang**: A concurrent functional programming language that powered the telecom industry and is still in use <a class="yt-timestamp" data-t="00:11:19">[00:11:19]</a>.
*   **Ada**: A general-purpose language named after Ada Lovelace (considered the world's first computer programmer), popular in the 1980s and still used by the Department of Defense <a class="yt-timestamp" data-t="00:11:26">[00:11:26]</a>.
*   **Prolog**: Pioneered logic programming <a class="yt-timestamp" data-t="00:11:38">[00:11:38]</a>.
*   **Meta Language**: Pioneered the polymorphic type system used by other statically typed functional languages like Haskell <a class="yt-timestamp" data-t="00:11:41">[00:11:41]</a>.

## Esoteric Languages

This realm includes rare and bizarre languages that are more like works of art than engineering tools <a class="yt-timestamp" data-t="00:11:51">[00:11:51]</a>. They often push boundaries or are designed for specific, non-practical purposes.

*   **INTERCAL (Compiler Language With No Pronounceable Acronym)**: The first known esoteric language (1972), designed as a parody of languages like Algol and Fortran, featuring paradoxical manuals and keywords like "please" and "mingle" <a class="yt-timestamp" data-t="00:11:58">[00:11:58]</a>.
*   **Brainfuck**: Known for its extreme minimalism, created by Urban Mueller. It works by initializing an array, providing a pointer, and eight characters to manipulate memory, resulting in very difficult-to-read code <a class="yt-timestamp" data-t="00:12:20">[00:12:20]</a>.
*   **Malbolge**: Named after the eighth circle of hell in Dante's Inferno, this language is notoriously difficult to program in <a class="yt-timestamp" data-t="00:12:35">[00:12:35]</a>.
*   **Chef**: A stack-based language designed to make code look like a cooking recipe, using sentences like "put ingredient into mixing bowl" <a class="yt-timestamp" data-t="00:12:51">[00:12:51]</a>.
*   **Shakespeare Programming Language**: Makes code resemble a Shakespearean play, providing low-level assembly control with 16th-century poetic verbosity <a class="yt-timestamp" data-t="00:13:11">[00:13:11]</a>.
*   **Piet**: Named after Piet Mondrian, it's stack-based and uses patterns of 20 different colors on a bitmap image to represent code, resembling abstract art <a class="yt-timestamp" data-t="00:13:21">[00:13:21]</a>.
*   **LOLCODE**: Provides a developer experience similar to an "lolcat" meme, with program boundaries like "HI" and "K THX BYE," and loops using "I'M IN YR" <a class="yt-timestamp" data-t="00:13:34">[00:13:34]</a>.
*   **EmojiCode**: A language where the syntax is entirely based on emojis, designed to streamline coding for developers who use many emojis in documentation <a class="yt-timestamp" data-t="00:13:52">[00:13:52]</a>.
*   **C-- (C-minus-minus)**: Designed as a portable assembly language that borrows heavily from C but omits many of its features <a class="yt-timestamp" data-t="00:14:11">[00:14:11]</a>.
*   **Holy C**: Created by Terry A. Davis for TempleOS, an operating system written under divine direction. It works like C but is just-in-time compiled on the OS, allowing it to act like a scripting language interacting directly with the kernel <a class="yt-timestamp" data-t="00:14:21">[00:14:21]</a>.

## Lowest Levels of Programming

These represent the absolute lowest levels of learning for a software engineer, moving from human-readable code to the fundamental building blocks of computing <a class="yt-timestamp" data-t="00:14:38">[00:14:38]</a>.

*   **Assembly Language**: There are many variations, each corresponding directly to a CPU architecture (like x86 or ARM) <a class="yt-timestamp" data-t="00:14:43">[00:14:43]</a>. It uses simple commands to manipulate values on the CPU's registers <a class="yt-timestamp" data-t="00:14:55">[00:14:55]</a>.
*   **Machine Code**: This level involves working with ones and zeros (raw binary), usually represented in hexadecimal format <a class="yt-timestamp" data-t="00:15:02">[00:15:02]</a>. It requires intimate knowledge of the computer's architecture <a class="yt-timestamp" data-t="00:15:10">[00:15:10]</a>.
*   **Transistors and Logic Gates**: Beyond machine code, one encounters billions of transistors on a CPU, where a single transistor represents one bit (a one or zero) by controlling electricity flow through silicon <a class="yt-timestamp" data-t="00:15:16">[00:15:16]</a>. These transistors are organized into logic gates (like NOT, AND, OR), which perform simple logic operations to produce outputs from electrical inputs, billions of times per second <a class="yt-timestamp" data-t="00:15:28">[00:15:28]</a>.
*   **Quantum Electrodynamics**: The deepest theoretical level, involving understanding how particles behave in the electromagnetic quantum vacuum, which could lead to building next-generation quantum computers <a class="yt-timestamp" data-t="00:15:49">[00:15:49]</a>.
