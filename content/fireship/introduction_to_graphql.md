[[GraphQL Query Language and Schema | GraphQL]] is a query language for APIs <a class="yt-timestamp" data-t="00:00:00">[00:00:00]</a>. It functions as a language that enables front-end applications to communicate with back-end applications, acting as a "Rosetta Stone" for consistent data requests <a class="yt-timestamp" data-t="00:00:34">[00:00:34]</a>.

At its core, [[GraphQL Query Language and Schema | GraphQL]] involves defining a schema for your data and then telling [[GraphQL Query Language and Schema | GraphQL]] how to fetch and supply that schema <a class="yt-timestamp" data-t="00:00:03">[00:00:03]</a>. The server defines types for the available data and resolvers to fetch that data from a database or other data source <a class="yt-timestamp" data-t="00:01:09">[00:01:09]</a>.

## Core Concepts

*   **Query Language for APIs**: [[GraphQL Query Language and Schema | GraphQL]] allows front-end applications to request data from back-end applications <a class="yt-timestamp" data-t="00:00:34">[00:00:34]</a>.
*   **Schema Definition**: You define a schema for your data <a class="yt-timestamp" data-t="00:00:03">[00:00:03]</a>. The server defines types for the data available <a class="yt-timestamp" data-t="00:01:11">[00:01:11]</a>.
*   **Runtime for Queries**: On the server, [[GraphQL Query Language and Schema | GraphQL]] functions as a runtime for executing queries <a class="yt-timestamp" data-t="00:01:07">[00:01:07]</a>.
*   **Resolvers**: These are functions that actually fetch the data from a database or other data source <a class="yt-timestamp" data-t="00:01:13">[00:01:13]</a>.
*   **Strongly Typed System**: Everything on the server is strongly typed, which enables powerful tooling <a class="yt-timestamp" data-t="00:01:19">[00:01:19]</a>. When hovering over a field in a [[GraphQL Query Language and Schema | GraphQL]] query, its type is displayed <a class="yt-timestamp" data-t="00:02:01">[00:02:01]</a>. Built-in types include string and integer, and servers can define custom types <a class="yt-timestamp" data-t="00:02:07">[00:02:07]</a>. This ensures front-end developers always know the exact shape of the data to expect <a class="yt-timestamp" data-t="00:02:14">[00:02:14]</a>.
*   **Introspection**: Consumers or front-end applications can perform introspection to the backend to see what data is available <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a>.

# Introduction to GraphQL

[[GraphQL Query Language and Schema | GraphQL]] is a query language for APIs <a class="yt-timestamp" data-t="00:00:00">[00:00:00]</a>. It defines a schema for data and dictates how [[GraphQL Query Language and Schema | GraphQL]] fetches and supplies that schema <a class="yt-timestamp" data-t="00:00:03">[00:00:03]</a>. In its most fundamental sense, [[GraphQL Query Language and Schema | GraphQL]] is a language that enables front-end applications to communicate with back-end applications, acting as a consistent "Rosetta Stone" for requesting data, regardless of underlying programming language differences <a class="yt-timestamp" data-t="00:00:34">[00:00:34]</a>.

For example, a mobile application can send a [[GraphQL Query Language and Schema | GraphQL]] query to request data, and the query will have the same shape as the expected JSON response from the API <a class="yt-timestamp" data-t="00:00:53">[00:00:53]</a>. On the server, [[GraphQL Query Language and Schema | GraphQL]] also functions as a runtime for executing queries <a class="yt-timestamp" data-t="00:01:07">[00:01:07]</a>. The server defines types for the available data and resolvers to fetch that data from a database or other data source <a class="yt-timestamp" data-t="00:01:09">[00:01:09]</a>.

## Advantages of GraphQL

Because everything on the server is strongly typed, [[GraphQL Query Language and Schema | GraphQL]] provides remarkable tooling <a class="yt-timestamp" data-t="00:01:19">[00:01:19]</a>. For consumers using the API in a front-end application, introspection can be performed on the backend to see what data is available <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a>. When hovering over a field in a [[GraphQL Query Language and Schema | GraphQL]] query, it displays its type <a class="yt-timestamp" data-t="00:02:01">[00:02:01]</a>. While there are built-in types like string and integer, the server can also define its own custom types <a class="yt-timestamp" data-t="00:02:07">[00:02:07]</a>. This means a front-end developer will always know the exact shape of the data expected back from the server <a class="yt-timestamp" data-t="00:02:14">[00:02:14]</a>.
