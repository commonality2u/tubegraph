## Coding vs. Programming

It's crucial to understand the distinction between coding and programming <a class="yt-timestamp" data-t="00:01:00">[00:01:00]</a>:

*   **Coding** is the skill of translating logic into instructions for a computer by writing code <a class="yt-timestamp" data-t="00:01:02">[00:01:02]</a>. While an elite coder might type 100 words per minute of perfect C/C++ code in [[features_of_popular_code_editors_like_vi_vim_and_emacs | Vim]] from memory <a class="yt-timestamp" data-t="00:01:08">[00:01:08]</a>, large language models (LLMs) are making this skill obsolete, allowing even an 8-year-old to write code faster than a professional with 20 years of experience <a class="yt-timestamp" data-t="00:01:13">[00:01:13]</a>.
*   **Programming** is a broader skill, an art and a science that demands ingenuity, creativity, and good instincts gained through experience <a class="yt-timestamp" data-t="00:01:22">[00:01:22]</a>. The most effective programmers often spend more time removing code than writing it <a class="yt-timestamp" data-t="00:01:31">[00:01:31]</a>. Without understanding the underlying code, pure [[vibe_coding_mind_virus_phenomenon | vibe coding]] is unlikely to succeed <a class="yt-timestamp" data-t="00:01:33">[00:01:33]</a>.

One example of [[risks_and_potential_success_stories_of_vibe_coding | vibe coding gone wrong]] involved an indie hacker whose SaaS product was hacked and taken down, forcing him to seek his old job back <a class="yt-timestamp" data-t="00:01:37">[00:01:37]</a>. This highlights the dangers of not understanding the code being "vibed out" <a class="yt-timestamp" data-t="00:01:33">[00:01:33]</a>.

## Rules for Effective Vibe Coding

When executed properly, [[risks_and_potential_success_stories_of_vibe_coding | vibe coding can be successful]], with potential for billion-dollar businesses built entirely on "Vibes" using tools like Vzero, Bolt, and [[ai_tools_in_web_development | AI IDEs]] such as Cursor and Windserve <a class="yt-timestamp" data-t="00:02:04">[00:02:04]</a>. To "vibe max" your code effectively, follow these three rules:

### 1. Choose a Popular, Simple Tech Stack

LLMs are most effective at solving problems that have already been addressed on platforms like GitHub and Stack Overflow <a class="yt-timestamp" data-t="00:02:30">[00:02:30]</a>. Therefore, selecting a widely used and straightforward tech stack is ideal. For web development, common choices like React, Express, Tailwind, Redis, and Deno are recommended <a class="yt-timestamp" data-t="00:02:22">[00:02:22]</a>. While personal preference might lean towards frameworks like Svelte, or even Vue.js and Nuxt for good [[vibe_coding_mind_virus_phenomenon | vibe coding]] results, using obscure or cutting-edge frameworks can hinder your workflow <a class="yt-timestamp" data-t="00:02:37">[00:02:37]</a>.

### 2. Get Good at Git

This rule is non-negotiable <a class="yt-timestamp" data-t="00:02:49">[00:02:49]</a>. When [[impact_of_ai_tools_on_coding_and_developers | AI]] generates code, it can also delete working code, making it nearly impossible to prompt it back into existence <a class="yt-timestamp" data-t="00:02:52">[00:02:52]</a>. Discipline in using version control, such as Git, allows you to stash working code safely <a class="yt-timestamp" data-t="00:03:01">[00:03:01]</a>. Tools like Claude Code can help "vibe commit" your code, or you can learn Git manually <a class="yt-timestamp" data-t="00:03:06">[00:03:06]</a>.

### 3. Act as a Product Manager for Your AI

You are no longer just a coder; you are a product manager guiding the [[impact_of_ai_tools_on_coding_and_developers | AI]] <a class="yt-timestamp" data-t="00:03:17">[00:03:17]</a>. To be effective, you must:
*   Break down complex requirements into small, highly specific steps <a class="yt-timestamp" data-t="00:03:22">[00:03:22]</a>.
*   Provide proper context to solve problems, including recent documentation and images for UI design <a class="yt-timestamp" data-t="00:03:26">[00:03:26]</a>.

While this might seem like a "vibe killer," it ensures the LLM is deterministic and avoids unwanted creativity <a class="yt-timestamp" data-t="00:03:32">[00:03:32]</a>.
