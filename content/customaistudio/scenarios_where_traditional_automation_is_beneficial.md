## Characteristics of Traditional Automation
Traditional automation is characterized by:
*   **Predefined Workflows** It typically involves workflows with scripts that execute specific tasks consistently <a class="yt-timestamp" data-t="00:00:30">[00:00:30]</a>.
*   **Repetitive and Routine Tasks** The tasks are often repetitive and routine, involving clear, step-by-step processes where data and workflow steps do not change significantly over time <a class="yt-timestamp" data-t="00:00:37">[00:00:37]</a>.
*   **Limited Flexibility** It is limited to specific, repetitive tasks and does not improve over time; it performs the same action consistently <a class="yt-timestamp" data-t="00:02:05">[00:02:05]</a>.
*   **Fixed Context** Traditional automation operates with a fixed context, following its rule set based on the data it's given, without understanding its broader role in a process <a class="yt-timestamp" data-t="00:06:44">[00:06:44]</a>.
*   **Predefined Rules** Decision-making in traditional automation strictly adheres to predefined rules <a class="yt-timestamp" data-t="00:10:59">[00:10:59]</a>.

## Advantages of Traditional Automation
Traditional automation offers several benefits for suitable tasks:
*   **Stability** It provides stable and reliable execution of tasks <a class="yt-timestamp" data-t="00:16:37">[00:16:37]</a>.
*   **Predictability** Users know exactly what to expect, as it performs the same action every time <a class="yt-timestamp" data-t="00:16:39">[00:16:39]</a>.
*   **Ease of Implementation** For simple tasks, it is easier to implement and manage <a class="yt-timestamp" data-t="00:16:40">[00:16:40]</a>. Workflows can often be set up quickly, sometimes in a couple of hours <a class="yt-timestamp" data-t="00:16:44">[00:16:44]</a>.

## Use Cases and Applications
Traditional automation is ideal for straightforward, repetitive tasks that are not expected to change <a class="yt-timestamp" data-t="00:20:10">[00:20:10]</a>. Specific examples include:
*   **Simple Data Entry** Moving consistent data from one location to another where the source and destination are fixed and the data type doesn't vary <a class="yt-timestamp" data-t="00:11:16">[00:11:16]</a>.
*   **Generating and Sending Onboarding Documents** This can include documents with variables that are consistently placed <a class="yt-timestamp" data-t="00:11:18">[00:11:18]</a>.
*   **Rule-Based Chatbots** These provide predefined answers, picking up on keywords to deliver canned responses <a class="yt-timestamp" data-t="00:11:21">[00:11:21]</a>.
*   **Automatically Sorting Incoming Emails** This works well when there are rigid, predefined rules for how emails should be sorted <a class="yt-timestamp" data-t="00:12:02">[00:12:02]</a>.
*   **Extracting Data from Invoices** This involves taking consistent data from invoices and entering it into an accounting system <a class="yt-timestamp" data-t="00:13:12">[00:13:12]</a>.

Traditional automation is best suited for tasks where the process and data inputs are highly consistent and predictable, requiring no reasoning or adaptation to variations <a class="yt-timestamp" data-t="00:00:40">[00:00:40]</a>. If data inputs or workflows vary, even slightly (e.g., changes in casing or small process evolutions), traditional automation can break <a class="yt-timestamp" data-t="00:03:11">[00:03:11]</a>. This means it struggles with [[challenges_in_automating_enterprise_processes | scalability]] as any change requires manual updates <a class="yt-timestamp" data-t="00:02:45">[00:02:45]</a>. In such cases, teams might find themselves working around the automation rather than the automation working for them <a class="yt-timestamp" data-t="00:17:32">[00:17:32]</a>.
