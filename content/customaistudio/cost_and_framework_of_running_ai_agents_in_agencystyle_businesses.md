## What Are AI Agents? <a class="yt-timestamp" data-t="00:06:20">[00:06:20]</a>

[[understanding_and_implementing_ai_agents_in_businesses | AI agents]] are large language models (LLMs) with the ability to take action <a class="yt-timestamp" data-t="00:06:24">[00:06:24]</a>. Unlike chatbots that only draft emails or answer questions, agents can actually send emails, categorize inbound messages, or conduct research <a class="yt-timestamp" data-t="00:06:31">[00:06:31]</a>.

This "agency" is achieved by giving LLMs "tools" and teaching them how to use those tools <a class="yt-timestamp" data-t="00:07:05">[00:07:05]</a>.

## Foundational Components of the AI Agent Framework <a class="yt-timestamp" data-t="00:14:40">[00:14:40]</a>

To [[building_and_deploying_ai_agents | build reliable AI agents]], specific components are required:

1.  **Vector Database**: Essential for agents to have contextual, up-to-the-minute knowledge of the business <a class="yt-timestamp" data-t="00:14:54">[00:14:54]</a>. This 3D database relates tokens (e.g., four characters of a word) by associating a vector (a number) with each, enabling relational connections between data points <a class="yt-timestamp" data-t="00:15:12">[00:15:12]</a>. When files are added to a knowledge base, the data is vectorized and stored in this database <a class="yt-timestamp" data-t="00:16:01">[00:16:01]</a>.
2.  **Data Capture & RAG Automations**: Automations are set up to capture all data streaming in and out of the business and add it to the vector database, keeping agents constantly updated <a class="yt-timestamp" data-t="00:16:16">[00:16:16]</a>. Retrieval Augmented Generation (RAG) automations ensure agents can retrieve this information when needed <a class="yt-timestamp" data-t="00:16:32">[00:16:32]</a>.
3.  **Tools**: An agent becomes an agent only when given tools <a class="yt-timestamp" data-t="00:16:49">[00:16:49]</a>. Tools enable agents to complete tasks and workflows. Examples include sending emails, creating calendar events, retrieving documents, updating tasks, searching/scraping the web <a class="yt-timestamp" data-t="00:17:32">[00:17:32]</a>. These can be custom-made <a class="yt-timestamp" data-t="00:17:45">[00:17:45]</a>.
4.  **Prompts**: Prompt engineering is critical, providing agents with precise instructions <a class="yt-timestamp" data-t="00:17:50">[00:17:50]</a>. Strong prompts include:
    *   Clear objective <a class="yt-timestamp" data-t="00:18:04">[00:18:04]</a>.
    *   Context (role, necessary info) <a class="yt-timestamp" data-t="00:18:11">[00:18:11]</a>.
    *   Specific instructions <a class="yt-timestamp" data-t="00:18:22">[00:18:22]</a>.
    *   Output requirements <a class="yt-timestamp" data-t="00:18:27">[00:18:27]</a>.
    *   Examples, especially for content generation or filtering <a class="yt-timestamp" data-t="00:18:36">[00:18:36]</a>.
5.  **Integrations**: Agents should integrate with existing tech stacks, acting as "glue" between platforms and eventually replacing some <a class="yt-timestamp" data-t="00:18:53">[00:18:53]</a>. Key questions for integration: what data is needed, where is it, where should results be output, and which APIs are necessary <a class="yt-timestamp" data-t="00:19:22">[00:19:22]</a>.

## Designing AI Agent Teams: Workflows, Not Tasks <a class="yt-timestamp" data-t="00:23:41">[00:23:41]</a>

A common misunderstanding is equating [[building_and_deploying_ai_agents | AI agents]] with regular automations <a class="yt-timestamp" data-t="00:23:46">[00:23:46]</a>. [[understanding_and_implementing_ai_agents_in_businesses | AI agents]] are closer to virtual assistants, given a job description, instructions, and tools <a class="yt-timestamp" data-t="00:23:53">[00:23:53]</a>.

The key is to think in "workflows," not individual "tasks" <a class="yt-timestamp" data-t="00:24:10">[00:24:10]</a>. While a single agent *could* theoretically be given hundreds of tools for a massive prompt, current technology limitations mean it's more effective to [[designing_and_integrating_ai_agent_teams_to_automate_business_functions | build AI agent teams]] <a class="yt-timestamp" data-t="00:26:03">[00:26:03]</a>. Each agent in a team is responsible for one specific workflow and is given only the tools needed for that workflow <a class="yt-timestamp" data-t="00:26:25">[00:26:25]</a>.

### Example: Automating Lead Generation <a class="yt-timestamp" data-t="00:26:44">[00:26:44]</a>

To automate a marketing funnel and lead generation:
*   Identify human actions in the workflow: capturing lead info, researching, personalized follow-ups, qualifying interested leads, booking calls <a class="yt-timestamp" data-t="00:26:50">[00:26:50]</a>.
*   Assign each workflow to a dedicated agent:
    *   **Lead Enrichment Agent**: Searches the web for information on new leads entering the CRM and saves it <a class="yt-timestamp" data-t="00:27:44">[00:27:44]</a>.
    *   **Lead Nurturing Agent**: Crafts personalized messages (content, case studies, trainings) based on enriched data <a class="yt-timestamp" data-t="00:27:54">[00:27:54]</a>.
    *   **Inbox Management Agent**: Monitors the inbox for responses from leads <a class="yt-timestamp" data-t="00:28:15">[00:28:15]</a>.
    *   **Lead Qualification Agent**: Reads responses, answers specific questions, and books qualified calls on the calendar if interest is confirmed <a class="yt-timestamp" data-t="00:28:30">[00:28:30]</a>.

This team approach means human involvement is primarily in creating the initial funnel/content and handling the final sales call <a class="yt-timestamp" data-t="00:29:31">[00:29:31]</a>. The entire lead generation process from cold to booked call is handled autonomously by [[designing_and_integrating_ai_agent_teams_to_automate_business_functions | AI agents]] <a class="yt-timestamp" data-t="00:29:57">[00:29:57]</a>.
